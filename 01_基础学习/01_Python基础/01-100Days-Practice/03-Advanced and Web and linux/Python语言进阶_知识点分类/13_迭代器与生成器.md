## 13.1 协议（Protocol）

> Python 没有 `protocol` / `interface` 关键字，但有自己的一套方式来达到同样目的。

**"定义协议" = 给对象定规矩：你必须有哪些方法，才能算"这类东西"。**

Python 中用**魔术方法**表示协议。`__iter__` 和 `__next__` 就是**迭代器协议**。

---

## 13.2 迭代器协议

### 核心方法

| 方法 | 一句话 |
|:-----|:------|
| `__iter__` | "把你自己交出来给 for 循环用" — 只调一次，返回自身 |
| `__next__` | "给我下一个值" — 反复调用，直到抛出 `StopIteration` |

### 手写迭代器示例（斐波那契）

```python
class Fib:
    """斐波那契数列 — 手动实现迭代器"""
    
    def __init__(self, num):
        self.num = num           # 生成多少个数
        self.a, self.b = 0, 1   # 当前值、下一个值
        self.idx = 0             # 当前计数
    
    def __iter__(self):          # 第①步：返回迭代器自身
        return self
    
    def __next__(self):          # 反复调用
        if self.idx < self.num:
            val = self.a         # 取当前值
            self.a, self.b = self.b, self.a + self.b  # 计算下一个
            self.idx += 1
            return val
        raise StopIteration      # 没有了！通知 for 停止


for i in Fib(5):
    print(i, end=" ")          # 0 1 1 2 3
```

## 13.3 可迭代对象 vs 迭代器

| | 可迭代对象 (Iterable) | 迭代器 (Iterator) |
|:--|:---------------------|:------------------|
| **有** | `__iter__()` | `__iter__()` + `__next__()` |
| **职责** | "可以被遍历" | "负责具体怎么遍历" |
| **类比** | 一本书 | 书签（记住读到哪了） |
| **例子** | `list`, `dict`, `str` | `iter(list)` 的结果 |
| **能重复遍历？** | ✅ 可以多次从头来 | ❌ 只能用一次 |

```python
lst = [1, 2, 3]          # 列表是 Iterable
it = iter(lst)           # it 是 Iterator

list(it)                 # [1, 2, 3]  ✅ 正常
list(it)                 # []          ❌ 空的！已耗尽
```

## 13.4 生成器 — 迭代器的语法糖

### yield 关键字

```python
def fib(num):
    """生成器函数"""
    a, b = 0, 1
    for _ in range(num):
        a, b = b, a + b
        yield a              # ← 暂停并产出，下次从这里继续！
```

**yield vs return 的本质区别**：

| return | yield |
|:-------|:------|
| 结束并返回 | 暂停并返回（下次还能回来）|
| 一次性杯子，用了就扔 | 暂停键，按播放能继续 |

### 执行过程

```python
def count_down(n):
    while n > 0:
        yield n              # 暂停，返回 n
        n -= 1

gen = count_down(3)         # gen = 生成器对象
next(gen)                    # 3
next(gen)                    # 2
next(gen)                    # 1
next(gen)                    # StopIteration!
```

### yield 的三种用法

#### 用法1：只产出（普通生成器）
```python
def count_down(n):
    while n > 0:
        yield n              # 只产出，不接收
        n -= 1
```

#### 用法2：只接收（协程基础）
```python
def receiver():
    while True:
        x = yield            # yield 不产出任何值（None），只负责接收
        print(f"收到: {x}")

gen = receiver()
next(gen)                     # "预激"——让执行走到 yield 处停下
gen.send("hello")             # 输出: 收到: hello
gen.send("world")             # 输出: 收到: world
```

#### 用法3：既产出又接收（完整协程）— 流式平均值

```python
def calc_avg():
    """流式计算平均值 — 协程"""
    total, counter = 0, 0
    avg_value = None
    while True:
        value = yield avg_value       # ⭐ 右边=产出，左边=接收
        total, counter = total + value, counter + 1
        avg_value = total / counter


gen = calc_avg()
next(gen)                           # 预激
print(gen.send(10))                  # 10.0    ← 首次平均
print(gen.send(20))                  # 15.0    ← (10+20)/2
print(gen.send(30))                  # 20.0    ← (10+20+30)/3
```

**`value = yield avg_value` 双重含义**：

| 方向 | 含义 |
|:-----|:-----|
| `yield avg_value` | 向外产出当前的 avg_value |
| `value = yield ...` | 从外接收 `send()` 发来的数据赋给 value |

## 13.5 Python 中定义"接口"的三种方案

### 方案一：鸭子类型（Duck Typing）— 最推荐

```python
# "如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子"
class Duck:
    def fly(self): print("鸭子飞")

class Airplane:
    def fly(self): print("飞机飞")        # 不需要继承接口！

def make_it_fly(thing):
    thing.fly()                            # 直接调用，不检查类型

make_it_fly(Duck())                       # ✅
make_it_fly(Airplane())                   # ✅
# Whale 没有 fly() → 运行时才报 AttributeError
```

### 方案二：抽象基类 ABC — 需要正式约束时

```python
from abc import ABC, abstractmethod

class Flyable(ABC):                        # 用 ABC 代替 interface
    @abstractmethod
    def fly(self): pass

class Duck(Flyable):                      # 必须继承
    def fly(self): print("鸭子飞")

class Bird(Flyable):
    pass                                   # ❌ 忘记实现 → TypeError!

d = Duck()                                 # ✅
b = Bird()                                 # ❌ Can't instantiate abstract class
```

### 方案三：typing.Protocol（Python 3.8+）— 静态类型检查

```python
from typing import Protocol

class Flyable(Protocol):                   # 纯类型提示
    def fly(self) -> None: ...

def make_it_fly(thing: Flyable):           # mypy 会检查
    thing.fly()

class PaperPlane:
    def fly(self): print("纸飞机")

make_it_fly(PaperPlane())                 # mypy 通过！（结构子类型，无需继承）
```

### 三种方案对比

```
┌──────────────────────────────────────────────────────┐
│              如何在 Python 中定义"接口"？                │
├──────────────────┬───────────────────┬────────────────┤
│   鸭子类型        │   ABC 抽象基类      │ typing.Protocol│
│   不声明任何东西   │   class X(ABC)    │  class X(...)  │
│   有方法就直接用   │   @abstractmethod │  (纯类型提示)    │
│   运行时可能报错   │   实例化时就报错    │  mypy 静态检查  │
│   推荐日常使用     │   推荐 API 设计    │  推荐大型项目    │
└──────────────────┴───────────────────┴────────────────┘
```
