## 11.1 三层关系

```
元类 (Metaclass)  ──创建──▶  类 (Class)  ──创建──▶  实例 (Instance)
    type                        President          obj = President()
```

| 层级 | 类比 | 代码 |
|:-----|:-----|:-----|
| 元类 | 类的工厂 | `type` / `SingletonMeta` |
| 类 | 对象的模具 | `President` |
| 实例 | 产品 | `obj = President()` |

**一句话总结**：普通类负责创建对象，元类负责创建类本身。

## 11.2 type — 一切之母

所有类的默认元类都是 `type`。

```python
# 方式一：class 语法（语法糖）
class Foo:
    pass

# 方式二：等价于用 type 手动创建
Foo = type('Foo', (), {})        # (类名, 父类元组, 属性字典)
```

## 11.3 自定义元类实现单例

```python
import threading

class SingletonMeta(type):
    """自定义元类 — 继承自 type"""
    
    def __init__(cls, *args, **kwargs):
        """
        当类 President 被【定义/创建】时调用
        cls → 正在被创建的类（即 President 本身）
        """
        cls.__instance = None       # ① 给类挂上类属性，存唯一实例
        cls.__lock = threading.RLock()
        super().__init__(*args, **kwargs)
    
    def __call__(cls, *args, **kwargs):
        """
        当你写 President() 时触发
        相当于拦截了实例化过程
        """
        if cls.__instance is None:
            with cls.__lock:
                if cls.__instance is None:
                    cls.__instance = super().__call__(*args, **kwargs)
        return cls.__instance


# metaclass=SingletonMeta 的含义：
# 不要用默认的 type 来创建我，用 SingletonMeta 来创建我
class President(metaclass=SingletonMeta):
    """总统（单例类）"""
    pass
```

### 触发时机对比

| 方法 | 触发时机 | 作用 |
|:-----|:---------|:-----|
| `cls.__new__` | 实例化时 | 创建实例对象（分配内存） |
| `cls.__init__` | `__new__` 后 | 初始化实例属性 |
| **元类 `__call__`** | `ClassName()` 时 | 拦截整个实例化过程 |
| **元类 `__init__`** | class 定义时 | 干预类的创建 |

## 11.4 super() 的深入理解

### 普通类中的 super

```python
class Animal:
    def speak(self):
        print("动物叫")

class Dog(Animal):
    def speak(self):
        super().speak()          # 跳到 MRO 链的下一个类
        print("汪汪")
```

**`super()` = "跳到 MRO 链的下一个类"**

### 元类中的 super

```python
# 写法 A：默认
class Foo:
    pass
# Python 内部执行：Foo = type('Foo', (), {})

# 写法 B：指定元类
class President(metaclass=SingletonMeta):
    pass
# Python 内部执行：President = SingletonMeta('President', (), {})
#                              ^^^^^^^^^^^^
#                         用你的元类来创建！
```
