## 7.1 Big-O 时间复杂度

### 复杂度从快到慢

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

| 大 O | 名称 | 典型算法 | 实际感受 |
|:----:|------|---------|---------|
| `O(1)` | 常数时间 | 哈希表查找、数组索引 | 瞬间完成 |
| `O(log n)` | 对数时间 | 二分查找 | 数据量×10，只多几次操作 |
| `O(n)` | 线性时间 | 遍历列表、线性搜索 | 数据量翻倍，时间翻倍 |
| `O(n log n)` | 对数线性 | 归并排序、快速排序（平均）| 高效排序的标准 |
| `O(n²)` | 平方时间 | 冒泡排序、选择排序、嵌套循环 | 数据量大时很慢 |
| `O(2ⁿ)` | 指数时间 | 斐波那契递归（无缓存）、子集枚举 | 数据量>30就不可接受 |
| `O(n!)` | 阶乘时间 | 全排列生成、旅行商问题（暴力）| 数据量>10就很慢 |

### 性能对比演示

```python
import time

def measure(label, func):
    start = time.perf_counter()
    result = func()
    elapsed = time.perf_counter() - start
    print(f'{label}: {elapsed:.4f}s')
    return result

N = 100_0000

measure('O(n)', lambda: list(range(N)))
measure('O(n²) N=3000', lambda: [[i*j for j in range(3000)] for i in range(3000)])
```

## 7.2 Python 内置类型操作复杂度

> **面试常考点！**

### `list` 列表

| 操作 | 复杂度 | 说明 |
|------|:-----:|------|
| `append(x)` 尾部添加 | **O(1)** | 最常用，推荐 |
| `pop()` 弹出尾部 | **O(1)** | |
| `insert(i, x)` 插入 | **O(n)** | 需要移动后续元素 |
| `pop(i)` 弹出中间 | **O(n)** | 需要移动后续元素 |
| `x in list` 搜索 | **O(n)** | 遍历查找 |
| `list[i]` 索引访问 | **O(1)** | 直接寻址 |

### `dict` 字典

| 操作 | 复杂度 | 说明 |
|------|:-----:|------|
| `d[key] = value` 设置 | **O(1)** 平均 | 哈希映射 |
| `d[key]` 获取 | **O(1)** 平均 | 哈希映射 |
| `del d[key]` 删除 | **O(1)** 平均 | |
| `key in d` 判断存在 | **O(1)** 平均 | ⭐ **远优于列表的 `in`** |

### `set` 集合

| 操作 | 复杂度 | 说明 |
|------|:-----:|------|
| `add(x)` / `remove(x)` | **O(1)** 平均 | |
| `x in s` 成员判断 | **O(1)** 平均 | ⭐ **去重 + 查重的神器** |

> 📌 **实用结论**：
> - 判断元素是否存在：**用 set > 用 list**
> - 频繁在头部插入/删除：**用 deque > 用 list**
> - 键值对存储：**用 dict**（平均 O(1)）

## 7.3 五大算法思想

| 思想 | 核心思路 | 经典问题 | 适用场景 |
|------|---------|---------|---------|
| **穷举法** (Brute Force) | 尝试所有可能 | 百钱百鸡、密码破解 | 规模小、无规律的问题 |
| **贪婪法** (Greedy) | 每步选当前最优 | 找零、背包近似解、Dijkstra | 有最优子结构的问题 |
| **分治法** (D&C) | 分→治→合并 | 归并排序、快速排序 | 可分解为独立子问题 |
| **回溯法** (Backtracking) | 试探→回退重选 | 八皇后、骑士巡逻、全排列 | 组合/排列/NP 问题 |
| **动态规划** (DP) | 记忆重叠子问题的解 | 斐波那契、爬楼梯、最长公共子序列 | 有重叠子问题+最优子结构 |

## 7.4 排序算法

### 手写排序代码

```python
# === 选择排序 O(n²) ===
def select_sort(items, comp=lambda x, y: x < y):
    """每次选择最小元素放到已排序区末尾"""
    items = items[:]          # 复制一份，不修改原列表
    for i in range(len(items) - 1):
        min_idx = i           # 假设当前位置是最小
        for j in range(i + 1, len(items)):
            if comp(items[j], items[min_idx]):
                min_idx = j   # 发现更小的，更新下标
        items[i], items[min_idx] = items[min_idx], items[i]  # 交换
    return items


# === 冒泡排序 O(n²) ===
def bubble_sort(items, comp=lambda x, y: x > y):
    """相邻元素比较交换，大的像气泡一样"浮"到末尾"""
    items = items[:]
    for i in range(len(items) - 1):
        swapped = False       # 优化：如果本轮没交换说明已经有序
        for j in range(len(items) - 1 - i):
            if comp(items[j], items[j + 1]):
                items[j], items[j + 1] = items[j + 1], items[j]
                swapped = True
        if not swapped:
            break             # 提前退出
    return items


# === 归并排序 O(n log n) ===
def merge(left, right, comp=lambda x, y: x < y):
    """合并两个有序列表"""
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if comp(left[i], right[j]):
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    # 将剩余元素追加到末尾
    result.extend(left[i:])
    result.extend(right[j:])
    return result


def _merge_sort(items, comp):     # 私有递归函数
    if len(items) < 2:
        return items
    mid = len(items) // 2
    left = _merge_sort(items[:mid], comp)
    right = _merge_sort(items[mid:], comp)
    return merge(left, right, comp)


def merge_sort(items, comp=lambda x, y: x < y):  # 公开接口
    return _merge_sort(list(items), comp)


# === 快速排序 O(n log n) 平均 ===
def quick_sort(items, comp=lambda x, y: x <= y):
    items = list(items)[:]
    _quick_sort(items, 0, len(items) - 1, comp)
    return items


def _quick_sort(items, start, end, comp):
    if start < end:
        pivot_pos = _partition(items, start, end, comp)
        _quick_sort(items, start, pivot_pos - 1, comp)
        _quick_sort(items, pivot_pos + 1, end, comp)


def _partition(items, start, end, comp):
    """分区: 选最后一个元素为枢轴，小的放左边，大的放右边"""
    pivot = items[end]
    i = start - 1
    for j in range(start, end):
        if comp(items[j], pivot):
            i += 1
            items[i], items[j] = items[j], items[i]
    items[i + 1], items[end] = items[end], items[i + 1]
    return i + 1
```

### 排序算法对比总结

| 算法 | 平均复杂度 | 最坏复杂度 | 是否稳定 | 额外空间 |
|------|:----------:|:----------:|:-------:|:--------:|
| 冒泡 Bubble | O(n²) | O(n²) | 是 ✅ | O(1) |
| 选择 Selection | O(n²) | O(n²) | 否 ❌ | O(1) |
| 插入 Insertion | O(n²) | O(n²) | 是 ✅ | O(1) |
| **归并 Merge** | **O(n log n)** | **O(n log n)** | **是 ✅** | O(n) |
| **快速 Quick** | **O(n log n)** | **O(n²)** | **否 ❌** | O(log n) |
| 堆 Heap Sort | O(n log n) | O(n log n) | 否 ❌ | O(1) |
| Timsort (Python内置)| O(n log n) | O(n log n) | 是 ✅ | O(n) |

> 📌 **实际使用**: 99% 的场景直接用 `sorted()` 或 `list.sort()` 即可！Python 使用 Timsort（归并+插入混合），非常高效。

## 7.5 查找算法

```python
# === 顺序查找 O(n) ===
def seq_search(items, key):
    """逐个遍历，找到返回下标，找不到返回 -1"""
    for index, item in enumerate(items):
        if item == key:
            return index
    return -1


# === 折半查找 O(log n) — 前提: 必须有序! ===
def bin_search(items, key):
    start, end = 0, len(items) - 1
    while start <= end:
        mid = (start + end) // 2      # 中间位置
        if key > items[mid]:          # 在右半边
            start = mid + 1
        elif key < items[mid]:         # 在左半边
            end = mid - 1
        else:                          # 找到了!
            return mid
    return -1                         # 不存在


# 测试
data = sorted([random.randint(1, 100) for _ in range(20)])
print(f'有序数据: {data}')
idx = bin_search(data, data[5])
print(f'bin_search 找到的位置: {idx}, 值: {data[idx]}')
```

## 7.6 经典算法例题

### 例题 1：两数之和 —— 哈希表优化

```python
# ❌ 暴力法 O(n²): 两层循环
def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return i, j
    return None

# ✅ 哈希表优化 O(n): 一遍遍历!
def two_sum_hash(nums, target):
    """
    核心思路: 遍历时记录"需要的补数"
    
    对于每个 num:
    1. 计算 complement = target - num
    2. 如果 complement 已经在字典中 → 直接找到答案!
    3. 否则把 {num: 下标} 存入字典继续
    """
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:       # 补数已经在前面出现过
            return seen[complement], i  # 返回两个下标
        seen[num] = i                # 记录当前值的位置
    return None


print(two_sum_hash([3, 8, 5, 12], 13))
# (0, 3) 因为 nums[0]=3 + nums[3]=12 = 13
# O(n) vs O(n²) 的巨大差距!
```

### 例题 2：有效括号 —— 栈的应用

```python
def is_valid_parentheses(s):
    """
    有效括号匹配 — 使用栈（后进先出 LIFO）
    
    思路: 遇到左括号入栈，遇到右括号则弹出检查是否匹配
    """
    stack = []                       # 栈
    pairs = {')': '(', '}': '{', ']': '['}  # 匹配规则

    for ch in s:
        if ch in pairs.values():     # 左括号 → 入栈
            stack.append(ch)
        elif ch in pairs.keys():     # 右括号 → 检查栈顶
            if not stack or stack[-1] != pairs[ch]:
                return False         # 栈空或不匹配 → 无效
            stack.pop()              # 匹配成功 → 弹出
        else:
            return False            # 非法字符

    return not stack                 # 栈空说明全部匹配成功


tests = ['()', '(())', '({[]})', '([)]', '(((']
for t in tests:
    print(f'{t:>10s} → {"有效" if is_valid_parentheses(t) else "无效"}')
#         () → 有效
#       (()) → 有效
#     ({[]}) → 有效
#       ([)] → 无效
#        ((( → 无效
```

### 例题 3：LRU 缓存淘汰策略

```python
from collections import OrderedDict

class LRUCache:
    """
    Least Recently Used 缓存 — 最近最少使用淘汰
    
    核心原理: 
    - OrderedDict 保持插入顺序
    - 被访问的移到末尾(最新)
    - 超容量时弹出头部(最久未使用的)
    """

    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)  # 访问过 → 移到最后(最新)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)  # 弹出头部的(最旧的)


cache = LRUCache(capacity=3)

for k, v in [('a', 1), ('b', 2), ('c', 3)]:
    cache.put(k, v)

print(cache.get('a'))   # 1, a 变最新
cache.put('d', 4)       # 超容量! 淘汰最久未使用的 b
print(f'缓存 keys: {list(cache.cache.keys())}')
# ['c', 'a', 'd']  — b 被淘汰了!
```
