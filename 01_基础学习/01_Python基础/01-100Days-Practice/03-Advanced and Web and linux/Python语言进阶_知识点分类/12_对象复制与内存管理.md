## 12.1 深拷贝 vs 浅拷贝

| | 浅拷贝 (Shallow Copy) | 深拷贝 (Deep Copy) |
|:--|:---------------------|:-------------------|
| **复制的是什么？** | 只复制容器本身 | 递归复制所有嵌套对象 |
| **内部元素** | 共享引用同一个对象 | 创建全新的独立副本 |
| `copy.copy()` | ✅ | ❌ |
| `copy.deepcopy()` | ❌ | ✅ |

### 示例

```python
import copy

a = [1, 2, [3, 4]]

b = copy.copy(a)           # 浅拷贝
c = copy.deepcopy(a)       # 深拷贝

a[2].append(99)

print(b)                  # [1, 2, [3, 4, 99]]  ← 受影响！
print(c)                  # [1, 2, [3, 4]]       ← 不受影响！
```

### 图解

```
浅拷贝:                          深拷贝:
  a ──┐                           a ──→ [1, 2, ─┐]
      ↓                                         │
    [1,2,┐]                 c ──→ [1, 2, ─┤]
       ↓                                ↓
  b ──→ [1,2,┐]              各自独立 → [3, 4]
           ↓                    ↘
         [3,4] ← 共享！        [3,4] ← 完全独立！
```

**一句话总结**：浅拷贝是"外壳新、里子旧"；深拷贝是"连皮带肉全新"。

## 12.2 垃圾回收机制 — 引用计数（主策略）

Python 每个对象内部都有一个**引用计数器** `ob_refcnt`：

### 引用计数 +1 的场景（4种）

| 场景 | 示例 | 解释 |
|:-----|:-----|:-----|
| 创建对象 | `a = 23` | 新对象诞生，计数=1 |
| 赋值引用 | `b = a` | 多了一个名字指向它 |
| 传入函数 | `f(a)` | 函数参数也是一次引用 |
| 存入容器 | `list1 = [a, a]` | 列表内部也持有引用 |

### 引用计数 -1 的场景（5种）

| 场景 | 示例 | 解释 |
|:-----|:-----|:-----|
| del 显式删除 | `del a` | 断开引用 |
| 重新赋值 | `a = 24` | 原来的23少了一个引用 |
| 离开作用域 | 函数执行完毕 | 局部变量自动销毁 |
| 容器被销毁 | `del list1` | 容器内元素的引用都-1 |
| 从容器移除 | `list.remove(a)` | 少了一个持有者 |

当引用计数降到 **0** 时，Python 立即回收该对象的内存。

```python
import sys
a = "hello"
print(sys.getrefcount(a))          # 2   （a 自己 + getrefcount 的临时参数）
b = a
print(sys.getrefcount(a))          # 3   （多了一个 b 引用）
del b
print(sys.getrefcount(a))          # 2   （回到原样）
```

## 12.3 循环引用问题 — 致命缺陷

```
list1 对象 ──────→ list2 对象
  ↑__________________↓
     互相引用！
```

两个对象的引用计数都是 1（互相引用），**永远不会降为 0** → **内存泄漏**！

```python
list1 = []
list2 = []
list1.append(list2)            # list1 引用了 list2
list2.append(list1)            # list2 又引用了 list1
del list1                      # 删除外部引用
del list2                      # 删除外部引用
# 内存泄漏！两个对象无法被回收
```

---

## 12.4 Python 的解决方案（两套辅助机制）

### 方案一：标记-清除（Mark-Sweep）

定期扫描所有对象：
1. 从根对象（全局变量、栈上局部变量）出发
2. 能遍历到的 → 标记为"存活"
3. 遍历不到的 → 即使引用计数 > 0，也视为垃圾 → 回收

### 方案二：分代收集（Generational Collection）

基于经验规律：**大多数对象生命周期都很短**。

```
第0代（新生代）: 新建的对象放这里，频繁检查垃圾
    ↓ 存活后晋升
第1代（中年代）: 检查频率降低
    ↓ 继续存活
第2代（老年代）: 很少检查（大概率会一直活着）
```

## 12.5 弱引用（weakref）— 手动避免循环引用

如果两个对象需要互相知道对方的存在，但又不想造成循环引用：

```python
import weakref

class Node:
    def __init__(self, value):
        self.value = value
        self.other = None

a = Node("A")
b = Node("B")

# 强引用会导致循环:
#   a.other = b;  b.other = a   💀 内存泄漏风险

# 弱引用方案 — 不增加引用计数！
a.other = weakref.ref(b)        # b 的弱引用
b.other = weakref.ref(a)        # a 的弱引用

# 使用时通过 .() 调用获取真实对象
print(a.other().value)           # 输出 "B"

del a, b                         # 两个对象都能正常回收了 ✓
```

### 强引用 vs 弱引用

```
强引用循环:                     弱引用循环:
  A ──强──→ B                    A ──强──→ B
  ↑_______↓                      A ←─弱── B
  refcnt=1 永不归零               refcnt 都能归零 ✓
```

**核心区别**：弱引用只是"知道对方在哪"，但不"抓住不放"，不影响对方的引用计数。

## 12.6 总结脑图

```
对象复制
 ├── 浅拷贝: 复制外壳，内部共享
 └── 深拷贝: 完全独立的新副本

垃圾回收
 ├── 主力: 引用计数（+1/-1 规则）
 ├── 缺陷: 循环引用导致内存泄漏
 └── 补救:
      ├── 标记-清除: 定期扫描不可达对象
      ├── 分代收集: 分代管理，新生代频繁检查
      └── 弱引用: 开发者手动避免循环引用
```
