## 6.1 LEGB 变量查找顺序

Python 搜索变量时按以下优先级（从高到低）：

```
L → Local        （局部变量，当前函数内）
E → Embedded      （嵌套作用域，外层函数）
G → Global        （全局变量，模块级别）
B → Built-in      （内置变量，如 print/len/True）
```

## 6.2 global 关键字

```python
counter = 0

def inc():
    global counter           # 声明使用全局变量
    for _ in range(100000):
        counter += 1
```

**作用**：在函数内部**声明或定义**全局变量。要么直接用现有的全局变量，要么创建一个新的放到全局作用域。

---

## 6.3 nonlocal 关键字

```python
def outer():
    x = 10
    def inner():
        nonlocal x          # 声明使用嵌套作用域的变量
        x = 20
    inner()
    print(x)                # 20（被修改了！）
```

**作用**：声明使用**嵌套作用域**中的变量（该变量必须已存在，否则报错）。

| 关键字 | 作用域 | 用途 |
|:------:|:-------|:-----|
| `global` | 全局作用域 | 跨函数访问/修改模块级变量 |
| `nonlocal` | 嵌套作用域 | 在闭包中修改外层函数的变量 |

## 6.4 闭包原理

### 什么是闭包？

闭包 = **函数 + 该函数定义时所处的环境变量**

```python
def record_time(func):       # 外层函数
    """装饰器函数"""
    def wrapper(*args, **kwargs):   # 内层函数 — 闭包
        start = time()
        result = func(*args, **kwargs)
        print(f'{func.__name__}: {time() - start}秒')
        return result
    return wrapper               # 返回内层函数
```

**关键点**：
- `wrapper` 函数引用了外层的 `func` 变量
- 即使 `record_time()` 执行完毕，`func` 也不会被销毁
- 因为 `wrapper` 仍持有对它的引用

### 闭包应用场景

| 场景 | 示例 |
|:-----|:-----|
| **装饰器** | `record_time(func)` 保存被装饰的函数引用 |
| **单例模式** | `instances = {}` 字典保存在闭包中 |
| **工厂函数** | 根据参数生成不同配置的函数 |

### 单例装饰器中的闭包示例

```python
from functools import wraps

def singleton(cls):
    instances = {}              # ← 闭包自由变量（字典缓存）
    
    @wraps(cls)
    def wrapper(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)   # 缓存实例
        return instances[cls]                        # 返回缓存的实例
    
    return wrapper
```

这里 `instances` 字典就是通过闭包机制"活"下来的——即使 `singleton()` 调用结束，`wrapper` 仍能访问它。
