## 15.1 SOLID 设计原则

| 原则 | 全称 | 含义 |
|:-----|:-----|:-----|
| **S**RP | Single Responsibility Principle | 单一职责 — 一个类只做该做的事（高内聚）|
| **O**CP | Open-Closed Principle | 开闭原则 — 对扩展开放，对修改关闭 |
| **L**SP | Liskov Substitution Principle | 里氏替换 — 子类可替换父类对象 |
| **I**SP | Interface Segregation Principle | 接口隔离 — 接口小而专，不大而全 |
| **D**IP | Dependency Inversion Principle | 依赖倒转 — 面向抽象编程（弱类型语言中被弱化）|

> **加粗字母合称 SOLID 原则**

### 其他重要原则

| 原则 | 全称 | 含义 |
|:-----|:-----|:-----|
| **CARP** | Composite/Aggregation Reuse Principle | 合成聚合复用 — **优先使用强关联关系而非继承**复用代码 |
| **LoD** | Law of Demeter (迪米特法则) | 最少知识原则 — 不要给没有必然联系的对象发消息 |

## 15.3 实战：策略模式 — 可插拔的哈希算法

```python
class StreamHasher:
    """哈希摘要生成器 — 策略模式"""
    
    def __init__(self, alg='md5', size=4096):
        self.size = size
        alg = alg.lower()
        # 动态获取 hashlib 中对应的算法类
        self.hasher = getattr(__import__('hashlib'), alg.lower())()
    
    def __call__(self, stream):
        return self.to_digest(stream)
    
    def to_digest(self, stream):
        """生成十六进制形式的摘要"""
        for buf in iter(lambda: stream.read(self.size), b''):
            self.hasher.update(buf)
        return self.hasher.hexdigest()


def main():
    hasher1 = StreamHasher()                    # 默认 MD5
    with open('Python-3.7.6.tgz', 'rb') as stream:
        print(hasher1.to_digest(stream))
    
    hasher2 = StreamHasher('sha1')              # 切换为 SHA1
    with open('Python-3.7.6.tgz', 'rb') as stream:
        print(hasher2(stream))


if __name__ == '__main__':
    main()
```

### 策略模式核心思想

通过参数动态切换算法实现，而不需要修改调用代码。这里：
- `alg='md5'` / `alg='sha1'` 就是策略的选择
- `getattr(__import__('hashlib'), alg)()` 动态获取对应算法类
- 调用方式完全不变 → 符合**开闭原则**
