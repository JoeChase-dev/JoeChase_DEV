## 8.1 三大支柱

| 支柱 | 含义 | Python 实现 |
|:-----|:-----|:-----------|
| **封装** | 隐藏内部细节，暴露接口 | 属性、方法、`_` 前缀约定 |
| **继承** | 复用已有类的代码 | `class Child(Parent)` |
| **多态** | 同一接口不同行为 | 方法重写、鸭子类型 |

## 8.2 抽象基类（ABC）

```python
from abc import ABCMeta, abstractmethod

# metaclass=ABCMeta → 声明该类是"抽象类"，不能直接实例化
# @abstractmethod → 标记"抽象方法"，子类必须实现
class Employee(metaclass=ABCMeta):
    """员工（抽象类）"""
    
    def __init__(self, name):
        self.name = name
    
    @abstractmethod                    # 强制子类必须实现
    def get_salary(self):
        """结算月薪（抽象方法）"""
        pass


class Manager(Employee):           # ✅ 实现了 get_salary → 可以实例化
    def get_salary(self):
        return 15000.0

class Programmer(Employee):
    def __init__(self, name, working_hour=0):
        self.working_hour = working_hour
        super().__init__(name)
    
    def get_salary(self):
        return 200.0 * self.working_hour


class Salesman(Employee):
    def __init__(self, name, sales=0.0):
        self.sales = sales
        super().__init__(name)
    
    def get_salary(self):
        return 1800.0 + self.sales * 0.05
```

**为什么需要抽象类？**
- 定义"通用规则/模板"，具体实现由子类决定
- `Employee("张三")` 会报错 — 不能直接创建
- 所有子类**必须实现** `get_salary()`，否则也报错
- 保证代码的规范性和一致性

## 8.3 枚举类型（Enum）

```python
from enum import Enum, unique

@unique                          # 禁止枚举值重复
class Suite(Enum):
    """花色"""
    
    SPADE, HEART, CLUB, DIAMOND = range(4)
    
    # 重载 "<" 运算符
    def __lt__(self, other):
        return self.value < other.value
```

### 枚举成员的属性

| 属性 | 类型 | 示例 |
|:-----|:-----|:-----|
| `.name` | `str` | `'SPADE'` — 成员名称 |
| `.value` | 赋值的类型 | `0` — 成员值 |

```python
s = Suite.SPADE
print(s.name)      # 'SPADE'
print(s.value)     # 0
```

### 字面常量 vs 符号常量 vs 枚举

| 方式 | 写法 | 评价 |
|:-----|:-----|:-----|
| 字面常量 | `if status == 0:` | ❌ 差评，含义模糊 |
| 符号常量 | `STATUS_OK = 1` | 👍 好评，语义清晰 |
| **枚举 Enum** | `class Status(Enum)` | ⭐ 最佳实践，带类型约束 |

## 8.4 工厂模式

```python
class EmployeeFactory:
    """创建员工的工厂（工厂模式 — 解耦合对象使用者与对象创建）"""
    
    @staticmethod
    def create(emp_type, *args, **kwargs):
        """创建员工"""
        all_emp_types = {'M': Manager, 'P': Programmer, 'S': Salesman}
        cls = all_emp_types[emp_type.upper()]
        return cls(*args, **kwargs) if cls else None


# 使用
emps = [
    EmployeeFactory.create('M', '曹操'),           # Manager
    EmployeeFactory.create('P', '荀彧', 120),       # Programmer
    EmployeeFactory.create('S', '典韦', 123000),     # Salesman
]
for emp in emps:
    print(f'{emp.name}: {emp.get_salary():.2f}元')
```

**核心价值**：调用者不需要知道具体创建了哪个类，只需传类型标识。

## 8.5 @property 属性装饰器

```python
class Poker:
    def __init__(self):
        self.index = 0
        self.cards = [...]
    
    @property                           # getter — 不需要加 ()
    def has_more(self):
        return self.index < len(self.cards)

poker = Poker()
print(poker.has_more)                 # True   ← 不用写 ()！
```

### 完整读写控制

```python
class Student:
    def __init__(self, score):
        self._score = score            # 下划线约定：内部属性
    
    @property                          # getter — 读取
    def score(self):
        return self._score
    
    @score.setter                      # setter — 设置（带校验）
    def score(self, value):
        if not 0 <= value <= 100:
            raise ValueError("成绩必须在0~100之间！")
        self._score = value


s = Student(85)
print(s.score)                        # 85     → 调用 getter
s.score = 95                          # ✅ 正常 → 调用 setter
s.score = 150                         # → ValueError!
```

**好处**：对外暴露"看起来像普通属性"的接口，内部可加入验证、计算、日志等逻辑。

## 8.6 实例化详解

### 基本语法

```python
class Player:
    def __init__(self, name):          # 构造函数
        self.name = name               # 实例属性
        self.cards = []

p1 = Player('张三')                   # 创建第1个实例
p2 = Player('李四')                   # 创建第2个实例
```

### 实例化过程拆解

```
p = Player('张三')

① 分配内存空间，创建空对象
② 自动调用 __init__ 方法
③ self 指向这个新对象
④ 执行 self.name = '张三'
⑤ self.cards = []
⑥ 返回对象引用给变量 p
```

### 多种创建方式

| 方式 | 代码 |
|:-----|:-----|
| 无参 | `d = Dog()` |
| 有参（位置） | `s = Student('张三', 20, 95)` |
| 有参（关键字） | `s = Student(score=95, name='张三', age=20)` |
| 批量到列表 | `[Student(n) for n in names]` |

## 8.7 __repr__ 与 __str__

| 方法 | 触发时机 | 面向谁 | 用途 |
|:-----|:---------|:------:|:-----|
| `__str__` | `print(obj)` / `str(obj)` / `f'{obj}'` | 用户 | 友好展示 |
| `__repr__` | `repr(obj)` / 交互式终端输入 obj | 开发者 | 调试信息 |

```python
class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score
    
    def __repr__(self):
        return f'Student({self.name}, {self.score})'     # 给开发者看
    
    def __str__(self):
        return f'{self.name}: {self.score}分'             # 给用户看


s = Student("张三", 95)
print(s)              # → 调用 __str__ → "张三: 95分"
repr(s)               # → 调用 __repr__ → "Student(张三, 95)"
```

**规则**：
- Python 优先使用 `__str__`
- 如果没定义 `__str__`，回退使用 `__repr__`

## 8.8 if __name__ == '__main__'

### 含义

| 运行方式 | `__name__` 的值 |
|:--------:|:---------------:|
| 直接运行该文件 | `'__main__'` |
| 被其他文件导入 | 模块名（文件名不含.py）|

### 为什么需要？

让同一个文件既能**独立运行**，又能安全地被**导入复用**。

```python
# utils.py
def add(a, b):
    return a + b

if __name__ == '__main__':
    print("我是被直接运行的！")
    print(add(3, 5))                # 直接运行时才执行

# demo.py
import utils
print(utils.add(10, 20))            # 导入时不执行 main 中的代码
```

**记忆口诀**："如果我是主角(`__main__`)，就开始表演(`main()`)"
