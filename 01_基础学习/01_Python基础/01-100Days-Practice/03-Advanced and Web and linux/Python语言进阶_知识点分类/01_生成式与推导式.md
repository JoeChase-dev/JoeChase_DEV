## 1.1 字典推导式

```python
prices = {
    'AAPL': 191.88,
    'GOOG': 1186.96,
    'IBM': 149.24,
    'ORCL': 48.44,
    'ACN': 166.89,
    'FB': 208.09,
    'SYMC': 21.29
}

# 生成新字典，过滤 value > 100 的
prices2 = {key: value for key, value in prices.items() if value > 100}
print(prices2)
# {'AAPL': 191.88, 'GOOG': 1186.96, 'IBM': 149.24, 'ACN': 166.89, 'FB': 208.09}
```

**语法格式**：`{key_expr: value_expr for item in iterable if condition}`

## 1.2 嵌套列表的坑

### ❌ 错误写法

```python
a = [[None] * 3] * 3
print(a)           # [[None, None, None], [None, None, None], [None, None, None]]
a[0][0] = 100
print(a)           # [[100, None, None], [100, None, None], [100, None, None]]
                    # 三行同时被修改了！
```

**原因**：`*` 复制的是引用，3 个子列表指向同一个对象！

### ✅ 正确写法

```python
names = ['关羽', '张飞', '赵云']
courses = ['语文', '数学', '英语']
score_list = [[None] * len(courses) for _ in range(len(names))]

for rows, name in enumerate(names):
    for cols, course in enumerate(courses):
        score_list[rows][cols] = float(input(f'请输入{name}的{course}成绩:'))
print(score_list)
```

**要点**：用列表推导式 `for _ in range()` 创建独立子列表，每个 `[None]*len(courses)` 都是新的列表对象。

## 1.3 列表推导式替代 map + filter

```python
# 传统写法（高阶函数）
items1 = list(map(lambda x: x ** 2, filter(lambda x: x % 2, range(1, 11))))

# 推导式写法（更 Pythonic）
items = [x ** 2 for x in range(1, 11) if x % 2]
```

**推导式优势**：
- 可读性更好
- 执行效率更高（CPython 有专门优化）
- Python 社区首选风格
