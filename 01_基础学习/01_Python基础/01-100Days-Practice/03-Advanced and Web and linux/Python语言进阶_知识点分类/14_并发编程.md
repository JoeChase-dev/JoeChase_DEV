## 14.1 概述

### 为什么需要并发？

| 好处 | 说明 |
|:-----|:-----|
| **提升性能** | 让没有因果关系的部分并发执行，充分利用 CPU 等待时间 |
| **改善体验** | 避免耗时操作导致程序"假死" |

### 三种方案对比

| 方案 | 适用场景 | 核心模块 | GIL 影响 |
|:-----|:---------|:--------:|:--------|
| **多线程** | I/O 密集型 + 需共享状态 | `threading` | 受 GIL 限制 |
| **多进程** | CPU 密集型计算 | `multiprocessing` | 绕过 GIL，利用多核 |
| **异步 I/O** | 高并发网络/大量 I/O | `asyncio` | 单线程，无 GIL 问题 |

## 14.2 进程 vs 线程

```
进程（Process）= 操作系统分配【内存】的基本单位
线程（Thread） = 操作系统分配【CPU】的基本单位
关系：一个进程可以包含一个或多个线程
```

---

## 14.3 GIL（全局解释器锁）

> **GIL = Global Interpreter Lock** — CPython 中防止多个线程同时执行字节码的互斥锁。

- 因为 CPython 内存管理非线程安全，GIL 导致 **Python 多线程无法真正并行利用多核 CPU**
- 对于 I/O 密集型任务影响不大（I/O 等待时会释放 GIL）
- 对于 CPU 密集型任务应使用**多进程**

## 14.4 多线程详解

### 基本用法

```python
import threading

threading.Thread(
    target='函数名',        # 目标函数
    args=('参数',)          # 位置参数元组
).start()
```

### 实战：多线程生成缩略图

```python
import glob
import os
import threading
from PIL import Image

PREFIX = 'thumbnails'

def generate_thumbnail(infile, size, format='PNG'):
    """生成缩略图"""
    file, ext = os.path.splitext(infile)
    file = file[file.rfind('/') + 1:]
    outfile = f'{PREFIX}/{file}_{size[0]}_{size[1]}.{ext}'
    img = Image.open(infile)
    img.thumbnail(size, Image.ANTIALIAS)
    img.save(outfile, format)

def main():
    if not os.path.exists(PREFIX):
        os.mkdir(PREFIX)
    for infile in glob.glob('images/*.png'):
        for size in (32, 64, 128):
            threading.Thread(
                target=generate_thumbnail,
                args=(infile, (size, size))
            ).start()

if __name__ == '__main__':
    main()
```

## 14.5 锁机制详解

### 锁的语法

```python
from threading import Lock, RLock

# Lock — 互斥锁
lock = Lock()

# 方式1：手动 acquire/release
lock.acquire()
try:
    # 临界区代码
finally:
    lock.release()

# 方式2：推荐用 with（自动释放）✅
with lock:
    # 临界区代码
```

### Lock vs RLock 对比

| 特性 | Lock | RLock |
|:-----|:----:|:-----|
| 可重入 | ❌ 同一线程再次获取会死锁 | ✅ 同一线程可多次 acquire |
| 计数器 | 无 | 有，acquire+1，release-1 |
| 适用场景 | 简单互斥 | 可能递归/嵌套调用的场景 |

### 核心作用

保证同一时刻只有一个线程执行某段代码（临界区），防止**竞态条件（Race Condition）**。

```python
# 竞态条件示例
counter = 0
def inc():
    global counter
    for _ in range(100000):
        counter += 1        # 非原子操作：读→改→写
# 多线程并发时可能同时读相同值，各自+1写回 → 结果 < 预期！

# 加锁后安全 ✅
lock = Lock()
def inc_safe():
    global counter
    for _ in range(100000):
        with lock:
            counter += 1   # 保证每次只有一个线程做 +=1
```

### 应用场景速查表

| 场景 | 推荐方案 |
|:-----|:--------|
| 共享变量更新 | Lock / RLock |
| 单例模式（线程安全）| RLock + 双重检查 |
| 文件/数据库写入 | Lock |
| 递归函数内加锁 | RLock |
| 生产者-消费者队列 | Lock + Condition |
| 资源池限制 | Lock / Semaphore |

### 选择原则

```
无重入需求 → 用 Lock（更轻量）
有递归/嵌套加锁 → 用 RLock
高并发读少量写 → 考虑读写分离 或 queue.Queue
```

## 14.6 Condition 条件变量（生产者-消费者模型）

> 当需要线程间协作调度（暂停/唤醒）时使用

### 三种同步工具对比

| 场景 | 解决方案 |
|:-----|:--------|
| 多线程竞争 1 个资源 | Lock / RLock |
| 多线程竞争多个资源 | Semaphore（信号量）|
| 多线程需要调度/协作 | **Condition（条件变量）**|

### Condition API

| 方法 | 作用 |
|:-----|:-----|
| `wait()` | 释放锁 → 挂起线程 → 被唤醒后重新获取锁 |
| `notify()` | 唤醒 1 个等待线程 |
| `notify_all()` | 唤醒所有等待线程 |

### 生产者-消费者完整示例

```python
from concurrent.futures import ThreadPoolExecutor
from random import randint
from time import sleep
import threading

class Account:
    def __init__(self, balance=0):
        self.balance = balance
        lock = threading.RLock()
        self.condition = threading.Condition(lock)   # Condition = 锁 + wait/通知机制
    
    def withdraw(self, money):           # 取钱 → 消费者
        with self.condition:
            while money > self.balance:          # ⭐ 用 while 不用 if！防虚假唤醒
                self.condition.wait()             # 释放锁 + 挂起当前线程
            new_balance = self.balance - money
            sleep(0.001)
            self.balance = new_balance
    
    def deposit(self, money):            # 存钱 → 生产者
        with self.condition:
            new_balance = self.balance + money
            sleep(0.001)
            self.balance = new_balance
            self.condition.notify_all()           # ⭐ 唤醒所有等待中的线程


def add_money(account):
    while True:
        money = randint(5, 10)
        account.deposit(money)
        print(threading.current_thread().name, ':', money, '====>', account.balance)
        sleep(0.5)

def sub_money(account):
    while True:
        money = randint(10, 30)
        account.withdraw(money)
        print(threading.current_thread().name, ':', money, '<====', account.balance)
        sleep(1)


def main():
    account = Account()
    with ThreadPoolExecutor(max_workers=15) as pool:
        for _ in range(5):
            pool.submit(add_money, account)         # 5个生产者
        for _ in range(10):
            pool.submit(sub_money, account)         # 10个消费者
```

**⚠️ 为什么用 while 不用 if？**
→ 防止**虚假唤醒（Spurious Wakeup）**，确保被唤醒后条件真正满足才继续执行。

## 14.7 线程池 ThreadPoolExecutor

```python
from concurrent.futures import ThreadPoolExecutor

pool = ThreadPoolExecutor(max_workers=10)      # 创建10个 worker 的线程池
future = pool.submit('函数名', '参数')           # 提交任务
pool.shutdown(wait=True)                         # 关闭线程池（等待完成）
result = future.result()                        # 获取返回值
```

### 完整示例：银行账户存取款竞争

```python
import time
import threading
from concurrent.futures import ThreadPoolExecutor

class Account(object):
    def __init__(self):
        self.balance = 0.0
        self.lock = threading.Lock()
    
    def deposit(self, money):
        with self.lock:                          # 通过锁保护临界资源
            new_balance = self.balance + money
            time.sleep(0.001)
            self.balance = new_balance

def main():
    account = Account()
    pool = ThreadPoolExecutor(max_workers=10)
    futures = []
    for _ in range(100):
        future = pool.submit(account.deposit, 1)
        futures.append(future)
    pool.shutdown()
    for future in futures:
        future.result()
    print(account.balance)                       # 100.0 ✅（无锁会 <100）
```

## 14.8 多进程详解

> 由于 GIL 的存在，多线程无法发挥 CPU 多核特性。对于计算密集型任务，使用多进程。

```python
import concurrent.futures
import math

PRIMES = [1116281, 1297337, 104395303, ...] * 5

def is_prime(n):
    """CPU 密集型：判断素数"""
    if n % 2 == 0:
        return False
    sqrt_n = int(math.floor(math.sqrt(n)))
    for i in range(3, sqrt_n + 1, 2):
        if n % i == 0:
            return False
    return True

def main():
    with concurrent.futures.ProcessPoolExecutor() as executor:
        for number, prime in zip(PRIMES, executor.map(is_prime, PRIMES)):
            print(f'{number} is prime: {prime}')
```

**效果验证**：如果 4 核 CPU 运行多进程程序，`user 时间 ≈ real 时间 × 4` → 成功利用了多核！

### 多线程 vs 多进程选择指南

| 使用多线程的场景 | 使用多进程的场景 |
|:----------------|:---------------|
| 维护许多共享状态（列表、字典等是线程安全的）| 执行计算密集型任务（数据处理、科学计算） |
| 大量时间花在 I/O 操作上 | 输入可并行分块，结果可合并 |
| 不需要太多内存，不依赖 I/O | 内存无限制且不强依赖 I/O |

## 14.9 异步 I/O（async/await）

> 用于高并发网络请求、大量 I/O 等待场景（爬虫、WebSocket、API 调用）

### 关键概念

| 关键字 | 作用 |
|:-------|:-----|
| `async def` | 定义协程函数 |
| `await` | 挂起当前协程，让事件循环去执行其他任务 |
| `asyncio.gather()` | 并发运行多个协程 |
| `asyncio.get_event_loop()` | 获取事件循环 |

### 基础示例

```python
import asyncio

async def prime_filter(m, n):
    primes = []
    for i in range(m, n + 1):
        # ... 判断素数逻辑 ...
        await asyncio.sleep(0.001)              # ⭐ 挂起，让出控制权给事件循环
    return tuple(primes)

async def square_mapper(m, n):
    squares = []
    for i in range(m, n + 1):
        squares.append(i * i)
        await asyncio.sleep(0.001)
    return squares

def main():
    loop = asyncio.get_event_loop()
    future = asyncio.gather(prime_filter(2, 100), square_mapper(1, 100))  # 并发运行
    loop.run_until_complete(future)
    loop.close()
```

### 异步 HTTP 请求示例（aiohttp）

```python
import asyncio
import re
import aiohttp

PATTERN = re.compile(r'\<title\>(?P<title>.*)\<\/title\>')

async def fetch_page(session, url):
    async with session.get(url, ssl=False) as resp:
        return await resp.text()

async def show_title(url):
    async with aiohttp.ClientSession() as session:
        html = await fetch_page(session, url)
        print(PATTERN.search(html).group('title'))

def main():
    urls = (
        'https://www.python.org/',
        'https://git-scm.com/',
        'https://www.jd.com/',
        'https://www.taobao.com/',
        'https://www.douban.com/'
    )
    loop = asyncio.get_event_loop()
    cos = [show_title(url) for url in urls]
    loop.run_until_complete(asyncio.wait(cos))       # 并发请求5个URL
    loop.close()
```

### 异步 I/O vs 多进程

当程序不需要真正的并行性，而是更多地依赖于**异步处理和回调**时，`asyncio` 是更好的选择。适合：
- 有大量等待与休眠
- 不需要实时数据处理
- Web 应用服务器
