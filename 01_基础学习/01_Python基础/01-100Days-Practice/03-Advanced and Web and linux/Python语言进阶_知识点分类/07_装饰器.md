## 7.1 基础装饰器

```python
from time import time

def record_time(func):
    """装饰器函数 — 记录函数执行时间"""
    def wrapper(*args, **kwargs):
        start = time()
        result = func(*args, **kwargs)       # 保存原函数返回值
        print(f'{func.__name__}: {time() - start}秒')
        return result                         # 返回原函数结果
    return wrapper


@record_time                              # 等价于 foo = record_time(foo)
def foo():
    ...
```

### 结构拆解

```
record_time (外层/工厂)
├── 参数: func（被装饰的函数）
├── 返回: wrapper（增强版函数）
│
└── wrapper (内层/替代者)
    ├── *args: 收集所有位置参数
    ├── **kwargs: 收集所有关键字参数
    └── 调用 func(*args, **kwargs) 执行原逻辑
```

**`*args` 和 `**kwargs` 的作用**：
- `*args` → 收集所有位置参数 `(1, 2, 3)`
- `**kwargs` → 收集所有关键字参数 `{name="张三", age=20}`

## 7.2 参数化装饰器

> 当装饰器需要接收自定义配置时使用（解耦输出方式）

```python
from functools import wraps
from time import time

def record(output):              # ← 多了一层，output 是参数
    """可以参数化的装饰器"""
    
    def decorate(func):          # ← 接收被装饰函数
        
        @wraps(func)             # ← 保留原函数元信息
        def wrapper(*args, **kwargs):
            start = time()
            result = func(*args, **kwargs)
            output(func.__name__, time() - start)   # 用传入的函数输出
            return result
        
        return wrapper
    
    return decorate               # ← 注意缩进！


# 使用方式
import logging
@record(logging.info)           # 把 logging.info 作为 output 传入
def my_func():
    pass
```

| 概念 | 含义 |
|:-----|:-----|
| 耦合 | 绑死了、分不开 |
| 解耦 | 让各部分独立，方便替换 |

## 7.3 类装饰器

```python
from functools import wraps
from time import time

class Record:
    """通过定义类的方式定义装饰器"""
    
    def __init__(self, output):
        self.output = output
    
    # 让类的实例能像函数一样被调用
    def __call__(self, func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            start = time()
            result = func(*args, **kwargs)
            self.output(func.__name__, time() - start)
            return result
        return wrapper


# 使用（和函数装饰器完全一样）
@Record(print)
def my_func():
    pass
```

**核心**：类装饰器靠 `__call__` 方法让实例变得可调用 `obj()`。

## 7.4 @wraps 装饰器的作用

```python
from functools import wraps

@wraps(func)     # 保留被装饰函数的元信息
def wrapper(...):
    ...
```

| 属性 | 不用 @wraps | 使用 @wraps |
|:-----|:------------|:-----------|
| `func.__name__` | `'wrapper'` | 原始名称 |
| `func.__doc__` | `wrapper` 的文档字符串 | 原始文档 |
| `func.__module__` | 当前模块 | 原始模块 |

**取消装饰器的魔法**：通过 `func.__wrapped__` 可以获取被装饰前的原始函数。

## 7.5 实战：单例模式装饰器

### 基础版（非线程安全）

```python
from functools import wraps

def singleton(cls):
    instances = {}                        # 类级别缓存字典（闭包变量）
    
    @wraps(cls)
    def wrapper(*args, **kwargs):
        if cls not in instances:          # 首次创建
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]             # 后续直接返回缓存
    
    return wrapper


@singleton                             # President = singleton(President)
class President:
    """总统类"""
    pass

p1 = President()
p2 = President()
print(p1 is p2)                       # True — 始终是同一个对象！
```

| 优点 | 缺点 |
|:-----|:-----|
| ✅ 代码简洁优雅 | ❌ 多线程不安全（可能创建多个实例）|
| ✅ 复用性强 | ❌ 子类继承需额外处理 |
| ✅ 对原有代码零侵入 | ❌ 无法再次初始化 |

### 线程安全版（双重检查锁定）

```python
from functools import wraps
from threading import RLock

def singleton(cls):
    instances = {}
    locker = RLock()                   # 可重入锁（闭包自由变量）

    @wraps(cls)
    def wrapper(*args, **kwargs):
        if cls not in instances:       # 第一重检查（外层，快速）
            with locker:               # 加锁
                if cls not in instances:  # 第二重检查（内层，安全）
                    instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    
    return wrapper
```

**为什么两重检查？**
- 外层 if：避免每次都加锁（性能优化）
- 内层 if：两个线程同时通过外层后，只有一个能创建实例
