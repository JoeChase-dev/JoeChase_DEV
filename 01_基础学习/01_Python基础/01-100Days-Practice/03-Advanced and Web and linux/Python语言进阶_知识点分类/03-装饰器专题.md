## 3.1 `@` 语法糖的本质

> `@decorator` 只是 Python 提供的"糖衣语法"，它的本质是：

```python
@simple_decorator
def say_hi():
    print("Hi!")

# ↑ 完全等价于 ↓

def say_hi():
    print("Hi!")
say_hi = simple_decorator(say_hi)  # 手动把函数传入装饰器，再赋值回去
```

### 最简单的装饰器

```python
def simple_decorator(func):
    """最基础的装饰器结构"""
    def wrapper():
        print("--- 函数执行前 ---")
        func()
        print("--- 函数执行后 ---")
    return wrapper

@simple_decorator
def say_hello():
    print("Hello!")

say_hello()
# 输出:
# --- 函数执行前 ---
# Hello!
# --- 函数执行后 ---
```

> 📌 **核心理解**：装饰器的本质是 **"函数接收函数，返回新函数"**。`@` 只是一个便捷的语法糖。

## 3.2 基础装饰器 —— 处理任意参数

> 实际使用中，被装饰的函数可能有各种参数，需要用 `*args` 和 `**kwargs` 来透传。

### 计时装饰器

```python
from time import time

def record_time(func):
    """计时装饰器 — 打印函数执行耗时"""
    
    def wrapper(*args, **kwargs):
        start = time()                    # 记录开始时间
        result = func(*args, **kwargs)     # 执行原函数，保留返回值
        end = time()                      # 记录结束时间
        print(f'{func.__name__}: {end - start:.4f}秒')
        return result                     # 返回原函数的结果
    
    return wrapper


# 使用方式一: @语法糖
@record_time
def add(a, b, c=0):
    return a + b + c

print(add(1, 2, c=3))   # add: 0.0000秒 → 输出 6


# 使用方式二: 手动调用（不修改原函数定义）
result = record_time(add)(1, 2, 3)
```

### 参数说明

| 参数 | 含义 | 示例 |
|------|------|------|
| `*args` | 收集所有**位置参数** | `func(1, 2, 3)` → `args = (1, 2, 3)` |
| `**kwargs` | 收集所有**关键字参数** | `func(a=1, b=2)` → `kwargs = {'a': 1, 'b': 2}` |

## 3.3 `functools.wraps` —— 修复元信息丢失 ⚠️

### 问题演示

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def important_function():
    """这个文档很重要！"""
    pass

# 没有 wraps 时:
print(important_function.__name__)  # 'wrapper' ❌ 变成了 wrapper 的名字！
print(important_function.__doc__)   # None ❌ 文档字符串丢失了！
```

### 解决方案

```python
import functools

def good_decorator(func):
    @functools.wraps(func)          # ✅ 保留原函数的元信息
    def wrapper(*args, **kwargs):
        """这是 wrapper 的文档，但不会覆盖原函数的"""
        return func(*args, **kwargs)
    return wrapper

@good_decorator
def important_function():
    """这个文档很重要！"""
    pass

# 有 wraps 时:
print(important_function.__name__)  # 'important_function' ✅
print(important_function.__doc__)   # '这个文档很重要！' ✅

# 还可以通过 __wrapped__ 取消装饰器，获取原始函数
original = important_function.__wrapped__
print(original.__name__)  # 'important_function'
```

> 📌 **知识点**：`@wraps(func)` 本身也是一个装饰器！它做了三件事：
> 1. 把原函数的 `__name__` 复制给 wrapper
> 2. 把原函数的 `__doc__` 复制给 wrapper  
> 3. 设置 `__wrapped__` 属性指向原函数（可用于取消装饰）

## 3.4 带参数的装饰器（三层嵌套）

> 当装饰器本身需要接受参数时，需要**三层嵌套**。

### 结构解析

```python
def 装饰器参数层(装饰器参数):           # 第1层：接收装饰器参数
    def 装饰器工厂(被装饰函数):         # 第2层：接收被装饰的函数
        @functools.wraps(被装饰函数)
        def 包装函数(*args, **kwargs):  # 第3层：执行包装逻辑
            # 这里可以使用 装饰器参数 和 被装饰函数
            result = 被装饰函数(*args, **kwargs)
            return result
        return 包装函数
    return 装饰器工厂
```

### 实战示例：可配置的计时器

```python
import functools
import time

def record(output):                       # 第1层: 接收装饰器参数 output
    """
    可参数化的装饰器
    
    Args:
        output: 一个回调函数，用于输出结果
                 例如: print、logging.info、或自定义函数
    """
    def decorate(func):                   # 第2层: 接收被装饰函数
        
        @functools.wraps(func)
        def wrapper(*args, **kwargs):     # 第3层: 执行包装逻辑
            start = time()
            result = func(*args, **kwargs)
            elapsed = time() - start
            output(func.__name__, elapsed)  # 用传入的 output 函数处理
            return result
        
        return wrapper
    return decorate                       # 返回装饰器工厂


# 使用方式一：用 print 作为输出方式
@record(output=lambda name, t: print(f'[PRINT] {name} 耗时 {t:.4f}s'))
def slow_function():
    time.sleep(0.1)
    return "done"

slow_function()
# [PRINT] slow_function 耗时 0.10xxs

# 使用方式二：自定义输出方式（如写入日志文件）
import logging
logger = logging.getLogger(__name__)

@record(output=lambda name, t: logger.info(f'{name}: {t:.4f}s'))
def another_slow_func():
    time.sleep(0.15)

another_slow_func()
```

### 三层嵌套记忆图

```
带参装饰器调用流程:

@repeat(times=3)
def greet():
    print("Hello!")

等价于:

greet = repeat(times=3)(greet)

         ┌─────────────────────────┐
第1步:     │ repeat(3)              │  ← 接收参数 times=3
         │   返回 decorate 函数    │
         └───────────┬─────────────┘
                     │
         ┌───────────▼─────────────┐
第2步:     │ decorate(greet)        │  ← 接收被装饰函数 greet
         │   返回 wrapper 函数     │
         └───────────┬─────────────┘
                     │
         ┌───────────▼─────────────┐
第3步:     │ greet = wrapper        │  ← 最终 greet 指向 wrapper
         │   每次调用执行3次       │
         └─────────────────────────┘
```

## 3.5 类装饰器 (`__call__`)

> 除了用函数做装饰器，还可以用**类**来实现——关键在于实现 `__call__` 方法。

### 基本写法

```python
class Record:
    """类装饰器：通过类的实例来装饰函数"""

    def __init__(self, output):
        self.output = output      # 存储装饰器参数

    def __call__(self, func):    # 让实例可像函数一样被调用 (@ 触发)
        
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            start = time.time()
            result = func(*args, **kwargs)
            elapsed = time.time() - start
            self.output(func.__name__, elapsed)  # 使用 self.output 输出
            return result
        
        return wrapper


# 使用方式和函数装饰器完全一样!
@Record(output=lambda name, t: print(f'[CLASS] {name}: {t:.4f}s'))
def compute(n):
    total = 0
    for i in range(n * 100000):
        total += i
    return total

compute(5)
# [CLASS] compute: 0.02xxx s
```

### 函数装饰器 vs 类装饰器对比

| 特性 | 函数装饰器 | 类装饰器 |
|------|----------|---------|
| 写法 | `def` + 内嵌函数 | `class` + `__call__` |
| 状态管理 | 用闭包变量 | 用 `self` 属性（更直观） |
| 复杂度 | 简单场景更简洁 | 复杂场景更有组织性 |
| 可读性 | 嵌套层级多时较难读 | 类结构更清晰 |

## 3.6 装饰器实战：单例模式（普通版）

> 单例模式确保一个类**只有一个实例**，并提供全局访问点。

```python
from functools import wraps

def singleton(cls):
    """
    单例装饰器 — 让被装饰的类只产生一个实例
    
    核心原理: 利用闭包的 instances 字典缓存已创建的对象
    """
    instances = {}                  # ① 类级别的缓存字典（闭包自由变量）

    @wraps(cls)
    def wrapper(*args, **kwargs):  # ② 替代原来的类
        if cls not in instances:   # ③ 如果还没创建过...
            instances[cls] = cls(*args, **kwargs)  # ④ 创建并缓存
        return instances[cls]      # ⑤ 后续直接返回缓存

    return wrapper


# 使用
@singleton
class President:
    """总统类 — 全局只能有一个总统"""
    pass


p1 = President()
p2 = President()

print(p1 is p2)  # True! ← 两个变量指向同一个对象
```

### 工作原理图解

```
第一次调用 President():
  wrapper() 检查 instances {}
  → 不在缓存中 → 创建对象 → 存入 instances {President: <obj>}
  → 返回 <obj>

第二次调用 President():
  wrapper() 检查 instances {President: <obj>}
  → 已在缓存中 → 直接返回 <obj>
  → p1 和 p2 是同一个对象!
```

### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✅ 代码简洁优雅 | ❌ **多线程环境下不安全** |
| ✅ 零侵入原类代码 | ❌ 子类继承需额外处理 |
| ✅ 复用性强（可装饰任何类） | ❌ 无法通过 `__new__` 再次初始化 |
| ✅ 利用闭包保持状态 | |

## 3.7 线程安全的单例（双重检查锁）

> 多线程环境下，普通单例可能会创建多个实例！解决方案：**加锁 + 双重检查**

### 为什么需要线程安全？

```
线程A: if cls not in instances? → True → （切换到B）→ 创建实例
线程B: if cls not in instances? → True → （切换回A）→ 也创建了实例!
结果: 出现了两个不同的实例 → 单例失效!
```

### 实现

```python
from functools import wraps
from threading import RLock

def singleton(cls):
    """线程安全的单例装饰器"""
    instances = {}                # 缓存字典
    locker = RLock()             # 可重入锁（也是闭包自由变量）

    @wraps(cls)
    def wrapper(*args, **kwargs):
        # 第一重检查（外层，无锁，快速判断）
        if cls not in instances:
            with locker:                          # 加锁
                # 第二重检查（内层，有锁，防止竞态）
                if cls not in instances:
                    instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    return wrapper
```

### 双重检查锁原理图解

```
线程A 进入 wrapper()
│
├── 第一重检查: cls not in instances? → True
│   ├── 加锁 locker (其他线程等待...)
│   │   ├── 第二重检查: cls not in instances? → True
│   │   │   └── 创建实例 ✓ (只有第一个进入的线程能到这里)
│   │   └── 释放锁
│   └── 返回缓存的实例
│
线程B 进入 wrapper() (稍晚)
│
├── 第一重检查: cls not in instances? → False (A已经创建了!)
│   └── 直接返回缓存的实例 (不需要加锁，性能更好!)
```

> 📌 **为什么用 RLock 而不是 Lock？**
>
> | 特性 | Lock | RLock |
> |------|------|------|
> | 同一线程重复 acquire | ❌ 死锁 | ✅ 允许（计数+1）|
> | 适用场景 | 简单互斥 | 递归/嵌套加锁 |
> | 推荐 | 无重入需求时 | ⭐ **有递归可能时** |
