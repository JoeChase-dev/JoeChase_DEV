# 第2月 W1：装饰器深入 + 生成器高级用法 详解版

## 第一部分：选择题答案解析

### 1. 带参数装饰器
**正确答案：A**  
`@repeat(num_times=3)`是一个带参数的装饰器，它返回真正的装饰器函数`decorator_repeat`。装饰器内部的`wrapper`函数虽然调用了3次`func`，但只返回最后一次的结果。因此输出是**单次**的`Hello Alice`。

**关键点**：带参数装饰器实际上是**三层嵌套函数**：最外层接受参数，中间层接受函数，最内层是包装器。

### 2. 类装饰器原理
**正确答案：C**  
类装饰器本质上是一个可调用对象，接受一个函数作为参数并返回包装后的函数。它通常需要实现`__call__`方法，但不一定需要`__init__`（如果无参数）。类装饰器也可以装饰类（如`@dataclass`）。

### 3. 装饰器链执行顺序
**正确答案：A**  
装饰器链的执行顺序是**从下往上装饰，从上往下执行**。`@decorator1`在最外层，`@decorator2`在内层。装饰时：先应用`decorator2`，再应用`decorator1`。执行时：先执行`decorator1`的`wrapper`，然后调用被装饰函数（此时已经是`decorator2`的`wrapper`），所以顺序是：Decorator 1 → Decorator 2 → Hello! → After decorator 2 → After decorator 1。

### 4. __wrapped__ 属性
**正确答案：B**  
`functools.wraps`的主要作用是保留被装饰函数的元信息（`__name__`、`__doc__`、`__module__`等），使装饰后的函数看起来更像原函数。它还设置了`__wrapped__`属性指向原函数。

### 5. 生成器send()方法
**正确答案：C**  
`gen.send(10)`将值`10`发送给生成器，`x = yield`接收这个值，打印`Received: 10`，然后执行到`yield x * 2`，返回`20`。注意：`print(result)`打印的是`yield x * 2`返回的值，即`20`，不是`Received: 10`。

### 6. yield from 协程
**正确答案：C**  
`yield from`会委托给子生成器，迭代子生成器的所有值，然后捕获子生成器的返回值。输出是：
```
1
2
Sub-generator returned: Done
```

### 7. 装饰器实现缓存
**正确答案：B**  
这是一个**记忆化（Memoization）**装饰器，将函数计算结果缓存起来，避免相同参数的重复计算。这是动态规划中常用的优化技术。

### 8. 生成器表达式 vs 列表推导
**正确答案：C**  
生成器表达式使用圆括号`()`，列表推导使用方括号`[]`。生成器表达式**惰性计算**，逐个生成元素，节省内存；列表推导**立即计算**，生成完整列表。

### 9. 带参数的类装饰器
**正确答案：C**  
类装饰器`Repeat(3)`创建实例，`__call__`方法返回包装函数。`wrapper`调用`func`3次，每次都会执行`print("Hi")`，所以输出3行`Hi`。

### 10. 装饰器中的闭包
**正确答案：B**  
Python的闭包机制允许内部函数访问外部函数的局部变量。`nonlocal`关键字用于在嵌套函数中修改外部函数的变量（Python 3+）。

### 11. 生成器状态机
**正确答案：A**  
`next(light)`输出`Red`，然后`yield color`暂停。`light.send("go")`恢复执行，但`command`是`"go"`，不是`"stop"`，继续循环到`Green`并输出。`light.send("stop")`触发`return`，生成器结束，抛出`StopIteration`。

### 12. 装饰器工厂模式
**正确答案：A**  
"装饰器工厂"指返回装饰器的函数，通常是带参数的装饰器的外层函数。例如`repeat(num_times)`返回真正的装饰器。

### 13. 生成器send()与初始化
**正确答案：A**  
生成器需要先用`next()`启动到第一个`yield`，然后才能`send()`。执行顺序：`next(g)`启动生成器，执行到`yield 1`，打印`Start`，返回`1`；`g.send(100)`发送`100`给`x`，打印`Got 100`，执行到`yield 2`，返回`2`。

### 14. 装饰器应用场景
**正确答案：B**  
装饰器最适合**横切关注点**：日志记录、性能监控、权限检查、缓存等。这些功能与业务逻辑分离，可以复用。

### 15. yield from 嵌套生成器
**正确答案：A**  
`yield from inner()`迭代子生成器的值`1, 2, 3`，然后捕获返回值`"Inner done"`，继续`yield f"Outer received: {result}"`。输出是：
```
1
2
3
Outer received: Inner done
```

## 第二部分：编程题参考实现

### 1. 实现带参数的装饰器
```python
import functools
import time

def retry(max_attempts):
    """带参数的重试装饰器"""
    def decorator_retry(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exception = None
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_exception = e
                    if attempt < max_attempts - 1:  # 不是最后一次尝试
                        time.sleep(1)
                        print(f"Attempt {attempt + 1} failed, retrying...")
            
            # 所有尝试都失败
            raise last_exception
        return wrapper
    return decorator_retry
```

### 2. 实现类装饰器
```python
import time
import functools

class Timer:
    """计时类装饰器"""
    def __init__(self, unit="seconds"):
        self.unit = unit
        if unit not in ["seconds", "milliseconds"]:
            raise ValueError("unit must be 'seconds' or 'milliseconds'")
    
    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            start_time = time.perf_counter()
            result = func(*args, **kwargs)
            elapsed = time.perf_counter() - start_time
            
            if self.unit == "milliseconds":
                elapsed *= 1000
                unit_str = "ms"
            else:
                unit_str = "s"
            
            print(f"Function '{func.__name__}' took {elapsed:.2f} {unit_str}")
            return result
        return wrapper
```

### 3. 生成器协程通信
```python
def running_average():
    """持续计算平均值的生成器"""
    total = 0
    count = 0
    
    while True:
        value = yield total / count if count > 0 else 0.0
        total += value
        count += 1

# 使用示例：
avg_gen = running_average()
next(avg_gen)  # 启动生成器
print(avg_gen.send(10))   # 输出: 10.0
print(avg_gen.send(20))   # 输出: 15.0
print(avg_gen.send(30))   # 输出: 20.0
```

### 4. 装饰器链实现
```python
import functools

def log_call(func):
    """记录函数调用的装饰器"""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Logging: {func.__name__} called with args {args}, kwargs {kwargs}")
        return func(*args, **kwargs)
    return wrapper

def validate_args(func):
    """验证参数类型的装饰器"""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        for arg in args:
            if not isinstance(arg, int):
                raise TypeError(f"Argument {arg} is not an integer")
        print(f"Validating args: {args} -> all integers")
        return func(*args, **kwargs)
    return wrapper

# 测试
@log_call
@validate_args
def add_numbers(a, b):
    return a + b

result = add_numbers(5, 3)
print(f"Result: {result}")
```

### 5. 复杂装饰器：缓存 + 过期时间
```python
import functools
import time
from typing import Any

def cached(ttl):
    """带过期时间的缓存装饰器"""
    def decorator_cached(func):
        cache = {}
        
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            # 生成缓存键（简化版，实际应用可能需要更复杂的键）
            cache_key = (func.__name__, args, frozenset(kwargs.items()))
            
            # 检查缓存
            if cache_key in cache:
                cached_result, timestamp = cache[cache_key]
                if time.time() - timestamp < ttl:
                    print(f"Cache hit for {func.__name__}{args}")
                    return cached_result
                else:
                    print(f"Cache expired for {func.__name__}{args}")
                    del cache[cache_key]
            
            # 计算新结果
            result = func(*args, **kwargs)
            cache[cache_key] = (result, time.time())
            print(f"Cache set for {func.__name__}{args}")
            
            return result
        return wrapper
    return decorator_cached
```

## 🔍 核心知识点总结

### 装饰器深入要点
1. **三层结构**：带参数装饰器 = 外层(参数) → 中层(函数) → 内层(包装器)
2. **类装饰器**：实现`__call__`方法，使其可调用
3. **装饰器链**：从下往上装饰，从上往下执行
4. **functools.wraps**：保留元信息，设置`__wrapped__`属性
5. **应用场景**：日志、缓存、权限、性能监控、参数验证等

### 生成器高级用法要点
1. **send()方法**：向生成器发送值，`x = yield`接收
2. **yield from**：委托给子生成器，捕获返回值
3. **生成器表达式**：`()`创建，惰性计算，节省内存
4. **协程风格**：生成器可作为简单协程，实现状态机
5. **初始化**：必须先`next()`或`send(None)`启动到第一个`yield`

### 常见陷阱与解决方案
1. **装饰器丢失元信息** → 使用`@functools.wraps`
2. **装饰器链顺序混乱** → 记住"从下往上装饰，从上往下执行"
3. **生成器未初始化就send** → 先`next(gen)`或`gen.send(None)`
4. **闭包变量修改** → 使用`nonlocal`关键字（Python 3+）
5. **缓存键冲突** → 使用`(func.__name__, args, frozenset(kwargs.items()))`
