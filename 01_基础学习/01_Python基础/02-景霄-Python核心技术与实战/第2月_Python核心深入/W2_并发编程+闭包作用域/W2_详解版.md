## 第一部分：选择题答案解析

### 1. GIL全局解释器锁
**正确答案：B**  
GIL是CPython解释器的特性，Jython（Java实现）和IronPython（.NET实现）没有GIL。GIL不保证线程安全（A错），它阻止多个线程同时执行Python字节码（C错）。多进程可以绕过GIL，但进程间通信有开销（D不完全正确）。

### 2. threading vs multiprocessing
**正确答案：B**  
CPU密集型任务受GIL限制，多线程无法真正并行。`multiprocessing`创建多个进程，每个进程有独立Python解释器和GIL，可以充分利用多核CPU。`asyncio`适合I/O密集型，单线程CPU任务。

### 3. 线程安全操作
**正确答案：D**  
Python中：
- `list.append()`**通常**线程安全（由于GIL和原子操作），但涉及resize时可能不安全
- `counter += 1`不是原子操作（读取-修改-写入），需要锁
- `dict[key] = value`在resize时可能不安全
严格来说都需要锁保证线程安全。

### 4. asyncio基础
**正确答案：B**  
`asyncio`通过事件循环+协程，在单线程中处理大量I/O操作，避免线程切换开销。不提高CPU任务性能（A错），不替代多进程（C错），不并行CPU任务（D错）。

### 5. 闭包变量捕获
**正确答案：B**  
闭包允许`inner()`函数访问`outer()`的局部变量`x`，即使`outer()`已返回。这是闭包的核心特性。

### 6. LEGB规则
**正确答案：B**  
LEGB = Local（局部）→ Enclosing（闭包）→ Global（全局）→ Built-in（内置）。这是Python查找变量的顺序。

### 7. nonlocal关键字
**正确答案：A**  
`nonlocal x`允许`inner()`修改`outer()`的`x`变量。每次调用`counter()`都会递增，输出1, 2。

### 8. 线程创建方式
**正确答案：B**  
正确方式：`thread = threading.Thread(target=func); thread.start()`。`run()`方法直接在当前线程执行，不创建新线程。

### 9. 进程间通信
**正确答案：D**  
全局变量在进程间不共享（每个进程有独立内存空间）。`Queue`、`Pipe`、`shared memory`是标准IPC方式。

### 10. asyncio事件循环
**正确答案：B**  
`asyncio.run()`自动创建新事件循环。每个线程最多一个事件循环（A错），可以在非主线程运行（C错），不并行CPU任务（D错）。

### 11. 闭包与装饰器关系
**正确答案：A**  
装饰器本质上是闭包：外层函数接受函数作为参数，内部函数包装原函数并返回。典型的闭包应用。

### 12. 线程池应用
**正确答案：D**  
代码正确。`ThreadPoolExecutor`适合I/O密集型任务（如网络请求），`map`会阻塞等待所有结果，上下文管理器自动关闭executor。

### 13. 协程async/await
**正确答案：A**  
`await asyncio.sleep(1)`暂停当前协程，让出控制权，1秒后恢复。输出：Hello → (等待1秒) → World。

### 14. 作用域链
**正确答案：A**  
遵循LEGB规则：
- `inner()`内：`x = "inner"` → 打印"inner"
- `outer()`内：`x = "outer"` → 打印"outer"  
- 全局：`x = "global"` → 打印"global"

### 15. 死锁避免
**正确答案：B**  
按固定顺序获取锁是避免死锁的基本策略（破坏循环等待条件）。其他选项可能有用，但不是根本解决方法。

### 1. 多线程爬虫
```python
import concurrent.futures
import time
import requests
from typing import List, Tuple

def download_page(url: str) -> Tuple[str, bool, float]:
    """下载单个网页，返回(URL, 是否成功, 耗时)"""
    start = time.perf_counter()
    try:
        response = requests.get(url, timeout=5)
        response.raise_for_status()  # 非200状态码抛异常
        elapsed = time.perf_counter() - start
        return url, True, elapsed
    except Exception as e:
        elapsed = time.perf_counter() - start
        print(f"Failed to download {url}: {e}")
        return url, False, elapsed

def multi_thread_crawler(urls: List[str], max_workers: int = 5):
    """多线程爬虫"""
    start_total = time.perf_counter()
    success_count = 0
    fail_count = 0
    
    with concurrent.futures.ThreadPoolExecutor(max_workers=max_workers) as executor:
        # 提交所有任务
        future_to_url = {executor.submit(download_page, url): url for url in urls}
        
        # 收集结果
        for future in concurrent.futures.as_completed(future_to_url):
            url, success, elapsed = future.result()
            if success:
                success_count += 1
                print(f"✓ {url} downloaded in {elapsed:.2f}s")
            else:
                fail_count += 1
                print(f"✗ {url} failed after {elapsed:.2f}s")
    
    total_time = time.perf_counter() - start_total
    
    print(f"\n{'='*40}")
    print(f"Successfully downloaded: {success_count} pages")
    print(f"Failed: {fail_count} pages")
    print(f"Total time: {total_time:.2f} seconds")
    print(f"Average time per page: {total_time/len(urls):.2f}s")
    return success_count, fail_count, total_time

# 测试
if __name__ == "__main__":
    urls = [
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/2",
        "https://httpbin.org/delay/3",
        "https://httpbin.org/status/404",  # 会失败
        "https://httpbin.org/delay/1"
    ]
    multi_thread_crawler(urls, max_workers=3)
```

### 2. 进程池计算
```python
import multiprocessing
import time
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n: int) -> int:
    """计算斐波那契数（递归实现，模拟CPU密集型）"""
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

def single_process_version(n: int, times: int = 10):
    """单进程版本"""
    start = time.perf_counter()
    results = []
    for _ in range(times):
        results.append(fib(n))
    elapsed = time.perf_counter() - start
    return results, elapsed

def multi_process_version(n: int, times: int = 10, processes: int = 4):
    """多进程版本"""
    start = time.perf_counter()
    
    # 创建进程池
    with multiprocessing.Pool(processes=processes) as pool:
        # 提交任务
        results = pool.map(fib, [n] * times)
    
    elapsed = time.perf_counter() - start
    return results, elapsed

# 测试
if __name__ == "__main__":
    n = 35
    times = 10
    
    print("单进程版本测试...")
    _, single_time = single_process_version(n, times)
    print(f"单进程耗时: {single_time:.2f}s")
    
    print("\n多进程版本测试（4进程）...")
    _, multi_time = multi_process_version(n, times, processes=4)
    print(f"多进程耗时: {multi_time:.2f}s")
    
    print(f"\n性能提升: {single_time/multi_time:.2f}x")
    print(f"时间节省: {single_time-multi_time:.2f}s")
```

### 3. asyncio并发请求
```python
import asyncio
import aiohttp
import time
from typing import List, Dict
import httpx  # 备选方案

async def fetch_url_aiohttp(url: str, session: aiohttp.ClientSession) -> Dict:
    """使用aiohttp下载网页"""
    try:
        async with session.get(url, timeout=10) as response:
            text = await response.text()
            return {"url": url, "status": response.status, "success": True}
    except Exception as e:
        return {"url": url, "error": str(e), "success": False}

async def fetch_all_urls(urls: List[str]) -> List[Dict]:
    """并发下载所有URL"""
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_url_aiohttp(url, session) for url in urls]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        return results

# 使用httpx的备选实现

        return {"url": url, "status": response.status_code, "success": True}

async def fetch_all_urls_httpx(urls: List[str]) -> List[Dict]:
```

### 4. 闭包实现计数器工厂
```python
def create_counter(initial: int = 0, step: int = 1):
    """
    创建计数器工厂
    
    Args:
        initial: 初始值
        step: 每次递增的步长
    
    Returns:
        计数器函数，每次调用返回递增后的值
    """
    count = initial  # 闭包变量
    
    def counter() -> int:
        nonlocal count
        current = count
        count += step
        return current
    
    return counter

# 测试
if __name__ == "__main__":
    # 创建两个独立的计数器
    counter1 = create_counter(initial=0, step=1)
    print("Counter 1:")
    print(counter1())  # 0
    print(counter1())  # 1
    print(counter1())  # 2
    
    counter2 = create_counter(initial=10, step=5)
    print("\nCounter 2:")
    print(counter2())  # 10
    print(counter2())  # 15
    print(counter2())  # 20
    
    # 验证独立性
    print("\nCounter 1 继续:")
    print(counter1())  # 3（独立于counter2）
    print(counter1())  # 4
    
    # 进阶：创建可重置的计数器
    def create_resettable_counter(initial=0, step=1):
        count = initial
        
        def counter(reset=False):
            nonlocal count
            if reset:
                count = initial
                return count
            current = count
            count += step
            return current
        
        return counter
    
    print("\n可重置计数器:")
    reset_counter = create_resettable_counter(initial=0, step=2)
    print(reset_counter())  # 0
    print(reset_counter())  # 2
    print(reset_counter())  # 4
    print(reset_counter(reset=True))  # 重置为0
    print(reset_counter())  # 2
```

### 5. 线程安全的数据结构
```python
import threading
import time
from typing import List
import concurrent.futures

class ThreadSafeCounter:
    """线程安全计数器"""
    
    def __init__(self, initial: int = 0):
        self._value = initial
        self._lock = threading.Lock()
    
    def increment(self, n: int = 1) -> int:
        """增加计数器值，返回新值"""
        with self._lock:
            self._value += n
            return self._value
    
    def decrement(self, n: int = 1) -> int:
        """减少计数器值，返回新值"""
        with self._lock:
            self._value -= n
            return self._value
    
    def get_value(self) -> int:
        """获取当前值"""
        with self._lock:
            return self._value
    
    def reset(self, new_value: int = 0) -> int:
        """重置计数器，返回重置后的值"""
        with self._lock:
            self._value = new_value
            return self._value
    
    def __str__(self) -> str:
        return f"ThreadSafeCounter(value={self.get_value()})"

# 测试函数
def worker(counter: ThreadSafeCounter, operations: int = 1000):
    """工作线程：交替执行增加和减少"""
    for i in range(operations):
        if i % 2 == 0:
            counter.increment()
        else:
            counter.decrement()

def test_thread_safety():
    """测试线程安全性"""
    counter = ThreadSafeCounter(initial=0)
    num_threads = 10
    operations_per_thread = 1000
    
    print(f"测试配置: {num_threads}个线程，每个{operations_per_thread}次操作")
    print(f"预期最终值: 0 (因为增加和减少次数相等)")
    
    start = time.perf_counter()
    
    # 使用线程池
    with concurrent.futures.ThreadPoolExecutor(max_workers=num_threads) as executor:
        futures = [executor.submit(worker, counter, operations_per_thread) 
                   for _ in range(num_threads)]
        
        # 等待所有线程完成
        concurrent.futures.wait(futures)
    
    elapsed = time.perf_counter() - start
    final_value = counter.get_value()
    
    print(f"\n测试结果:")
    print(f"最终值: {final_value}")
    print(f"耗时: {elapsed:.3f}秒")
    print(f"总操作数: {num_threads * operations_per_thread}")
    
    if final_value == 0:
        print("✅ 测试通过：计数器线程安全")
    else:
        print(f"❌ 测试失败：预期0，实际{final_value}")
    
    return final_value == 0

# 进阶：使用上下文管理器简化锁操作
class AtomicOperation:
    """原子操作上下文管理器"""
    
    def __init__(self, lock: threading.Lock):
        self._lock = lock
    
    def __enter__(self):
        self._lock.acquire()
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self._lock.release()

# 使用示例
if __name__ == "__main__":
    # 基础测试
    success = test_thread_safety()
    
    # 上下文管理器示例
    print("\n上下文管理器示例:")
    counter = ThreadSafeCounter()
    lock = threading.Lock()
    
    with AtomicOperation(lock):
        counter.increment(5)
        print(f"原子操作后: {counter.get_value()}")
```

## 🔍 核心知识点总结

### 并发编程要点
1. **GIL限制**：CPython中，GIL阻止多线程真正并行执行Python字节码
2. **适用场景**：
   - I/O密集型 → `threading`/`asyncio`
   - CPU密集型 → `multiprocessing`
3. **线程安全**：需要锁保护共享资源，即使有GIL也不能完全保证
4. **进程间通信**：Queue、Pipe、共享内存，但**不能**用全局变量

### asyncio要点
1. **事件循环**：单线程调度多个协程，高效处理I/O等待
2. **async/await**：`async def`定义协程，`await`暂停等待
3. **并发执行**：`asyncio.gather()`并发运行多个协程
4. **不适用场景**：CPU密集型任务（会阻塞事件循环）

### 闭包与作用域要点
1. **LEGB规则**：Python查找变量的顺序
2. **闭包本质**：内部函数记住并访问外部函数的变量
3. **nonlocal**：修改闭包变量（Python 3+）
4. **装饰器关系**：装饰器是闭包的典型应用

### 常见陷阱与解决方案
1. **GIL误解**：以为多线程绝对安全 → 理解GIL只保护字节码执行，不保护数据
2. **死锁**：多个锁获取顺序不一致 → 按固定顺序获取锁
3. **进程启动慢**：Windows下`multiprocessing`需要`if __name__ == "__main__"`
4. **协程阻塞**：在协程中执行CPU任务 → 使用`loop.run_in_executor()`
