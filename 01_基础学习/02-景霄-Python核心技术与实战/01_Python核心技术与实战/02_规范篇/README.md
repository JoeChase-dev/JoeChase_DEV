### 4️⃣ [pdb和cProfile](./04_pdb和cProfile.ipynb)

**学习目标**：
- 掌握pdb调试器的使用方法
- 学会使用cProfile进行性能分析
- 能够定位和修复代码中的bug和性能瓶颈

**核心内容**：
1. 为什么需要pdb？（vs print、vs IDE调试器）
2. pdb的基本使用（`set_trace()`、常用命令）
3. pdb常用命令详解：
   - `p`：打印变量
   - `n`：执行下一行
   - `s`：进入函数内部
   - `r`：执行到函数返回
   - `b`：设置断点
   - `l`：列出源代码
4. cProfile的基本用法（命令行和代码中使用）
5. cProfile输出结果解读（ncalls、tottime、cumtime）
6. 优化代码并重新profile

**练习题目**：2个（含详细答案）
- 使用pdb调试一个bug程序
- 使用cProfile分析斐波那契函数的性能

**预计学习时间**：2-3小时

## 💡 学习建议

### 学习方法
1. **理解规范的价值**：不要为了规范而规范，理解背后的设计思想
2. **多写单元测试**：单元测试是保证代码质量的关键
3. **熟练使用调试工具**：pdb和cProfile是必备技能
4. **实践上下文管理器**：在实际项目中应用，提高代码简洁度

### 重点和难点
- **重点**：单元测试（实际工作中使用最多）、编程规范（团队协作必备）
- **难点**：mock和patch的使用（需要多练习才能掌握）

### 时间规划
- **在校学习期**：每周完成1个主题（利用周末集中学习）
- **假期集训**：每天完成1个主题（连续4天）
