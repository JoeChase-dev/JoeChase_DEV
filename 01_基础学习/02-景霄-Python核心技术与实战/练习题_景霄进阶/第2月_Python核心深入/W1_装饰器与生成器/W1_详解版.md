### 🔘 第1题 — 装饰器基础 ⭐
以下代码的输出结果是什么？

**正确答案: B** ✅ → `Before Hello After`

```python
# 解析：
# @my_decorator 语法 = my_decorator(say_hello) 的语法糖
# 所以 say_hello 实际上指向了 wrapper 函数

def my_decorator(func):
    def wrapper():
        print("Before")   # ① 先执行前置代码
        func()             # ② 调用原始函数（打印 "Hello"）
        print("After")     # ③ 执行后置代码
    return wrapper

@my_decorator
def say_hello():
    print("Hello")

say_hello()
# 输出：
# Before
# Hello
# After
```

> 💡 **装饰器本质**：`@decorator` 等价于 `func = decorator(func)`，是一个**函数变换**操作。

### 🔘 第2题 — 装饰器执行时机 ⭐⭐
装饰器的核心执行时机是？

**正确答案: B** ✅ → **定义被装饰函数时（模块导入时）**

```python
# 证明：装饰器在导入时执行
def trace(func):
    print(f"✅ 装饰器执行了！装饰函数: {func.__name__}")
    def wrapper():
        return func()
    return wrapper

@trace
def foo():
    print("foo running")

# 运行上述代码，不用调用 foo()，就会看到：
# ✅ 装饰器执行了！装饰函数: foo
```

> - 装饰器在**模块导入时**执行（不是调用时）
> - 如果函数很多，导入会很慢（因为所有装饰器都会执行）
> - 有时需要**延迟装饰**，可以用 `functools.lru_cache` 之类的方式

### 🔘 第3题 — 带参数装饰器 ⭐⭐
实现一个可接收参数的装饰器（如 `@repeat(3)`），其嵌套层数应为？

**正确答案: C** ✅ → **3层（outer → decorator → wrapper）**

```python
# 三层嵌套结构：
# 第1层：接收装饰器参数
def repeat(n):
    # 第2层：接收被装饰的函数
    def decorator(func):
        # 第3层：包装函数，实现额外逻辑
        def wrapper():
            for _ in range(n):
                func()
        return wrapper
    return decorator

# 使用：
@repeat(3)        # ← 这里 repeat(3) 返回 decorator，然后 @decorator 装饰 func
def hello():
    print("Hello")
```

**记忆口诀**：
```
无参数装饰器：2层（decorator → wrapper）
带参数装饰器：3层（outer → decorator → wrapper）
```

### 🔘 第4题 — functools.wraps 作用 ⭐
使用 `functools.wraps(func)` 的主要目的是？

**正确答案: B** ✅ → **保留原始函数的元数据（__name__, __doc__ 等）**

```python
import functools

def my_decorator(func):
    def wrapper():
        """这是 wrapper 的文档"""
        return func()
    return wrapper

def my_decorator_proper(func):
    @functools.wraps(func)   # ← 关键！把 func 的元数据复制给 wrapper
    def wrapper():
        """这是 wrapper 的文档"""
        return func()
    return wrapper

@my_decorator
def foo():
    """这是 foo 的文档"""
    pass

@my_decorator_proper
def bar():
    """这是 bar 的文档"""
    pass

print(foo.__name__)   # 输出: wrapper  ❌（元数据丢失）
print(bar.__name__)   # 输出: bar     ✅（元数据保留）
print(foo.__doc__)    # 输出: 这是 wrapper 的文档  ❌
print(bar.__doc__)    # 输出: 这是 bar 的文档     ✅
```

> 📌 **最佳实践**：**所有装饰器的 wrapper 函数都必须加 `@functools.wraps(func)`！**

### 🔘 第5题 — 类装饰器 ⭐⭐
以下类装饰器的实现中，哪个方法必须定义？

**正确答案: B** ✅ → **`__call__`**

```python
class CountCalls:
    def __init__(self, func):
        self.func = func
        self.count = 0
    
    def __call__(self, *args, **kwargs):   # ← 必须定义，让实例可调用
        self.count += 1
        print(f"Call #{self.count}")
        return self.func(*args, **kwargs)

@CountCalls
def my_func():
    pass

# @CountCalls 等价于：my_func = CountCalls(my_func)
# 现在 my_func 是一个 CountCalls 实例，调用 my_func() 实际是调用 __call__()

my_func()   # Call #1
my_func()   # Call #2
```

> 💡 **类装饰器原理**：
> - `__init__`：接收被装饰的函数，保存为实例属性
> - `__call__`：让实例变得可调用（像函数一样使用）

### 🔘 第6题 — 装饰器链执行顺序 ⭐⭐⭐
```python
@decorator_A
@decorator_B
@decorator_C
def func():
    pass
```
等价于哪种写法？

**正确答案: A** ✅ → `decorator_A(decorator_B(decorator_C(func)))`

```python
# 装饰器链的等价写法（从下往上包裹）：
# @A
# @B
# @C
# def func(): pass

# 等价于：
func = A(B(C(func)))
#        ↑_____↑_____↑
#        从里到外依次包裹

# 执行顺序（调用 func() 时）：
# ① 先进入 A 的 wrapper
# ② A 的 wrapper 中调用 B 的 wrapper
# ③ B 的 wrapper 中调用 C 的 wrapper
# ④ C 的 wrapper 中调用原始 func
# ⑤ 返回时反向：C → B → A
```

**记忆口诀**：**"就近原则"** — 离函数定义最近的装饰器**最内层**，先装饰。

### 🔘 第8题 — yield 与 return 区别 ⭐

**正确答案: B** ✅ → **`return` 结束后函数终止，`yield` 暂停函数并保留状态**

```python
# return：函数结束，所有局部变量销毁
def with_return():
    print("before return")
    return 1
    print("after return")   # ← 永远不会执行

# yield：暂停函数，局部变量状态保留
def with_yield():
    print("before yield")
    yield 1
    print("after yield")    # ← 下次 next() 时会执行
    yield 2

g = with_yield()
next(g)   # 输出: before yield    （暂停在 yield 1）
next(g)   # 输出: after yield     （从暂停处继续，到 yield 2 暂停）
```

### 🔘 第9题 — send() 方法 ⭐⭐⭐
以下代码的输出是什么？

**正确答案: B** ✅ → `42`

```python
def coroutine():
    x = yield 1    # ① yield 1 给调用者；② 当调用者 send(42)，42 赋值给 x
    yield x         # ③ yield 接收到的值（42）

g = coroutine()
next(g)            # 启动生成器，执行到第一个 yield，产出 1（但没接收）
print(g.send(42))  # 把 42 发送给生成器（赋值给 x），继续执行到下一个 yield，产出 x=42
```

**send() 工作机制**：
```
生成器侧：x = yield value   ← yield 一个值，同时可以接收一个值
调用者侧：g.send(data)     ← 发送数据给生成器，同时收到 yield 的值
```

> ⚠️ **注意**：第一次启动生成器必须用 `next(g)` 或 `g.send(None)`，不能用 `g.send(数据)`！

```python
def trace(func):
```

### 🖥️ 第14题 — 生成器惰性求值 ⭐⭐

**输出：**
```
got generator
yield 1
got value: 1
```

**解析**：
```python
def gen():
    print("yield 1")    # ③ 第一次 next(g) 执行到这里
    yield 1              # ④ 产出 1，暂停
    print("yield 2")    # ⑥ 第二次 next(g) 才执行这里（本次没执行到）
    yield 2
    print("yield 3")
    yield 3

g = gen()
print("got generator")   # ① 立即执行（gen() 只是返回生成器对象，不执行函数体）
val = next(g)            # ② 驱动生成器执行到第一个 yield
print(f"got value: {val}")  # ⑤ 打印收到的值
# 注意："yield 2" 没有打印！因为只调用了一次 next(g)
```

> 💡 **惰性求值核心**：生成器函数体**不会在调用时执行**，只有 `next()` 或 `for` 循环驱动时才执行，且每次只执行到下一个 `yield`。

### 🖥️ 第15题 — send() 与 yield 交互 ⭐⭐⭐

**输出：**
```
Received: A
Received: B
```

**解析**：
```python
def echo():
    while True:
        received = yield    # ① 暂停在这里，等待接收数据
        print(f"Received: {received}")

g = echo()
next(g)        # 启动生成器，执行到第一个 yield 并暂停（此时还没接收数据）
g.send("A")   # 把 "A" 发送给生成器 → received = "A" → 打印 "Received: A" → 循环回到 yield
g.send("B")   # 把 "B" 发送给生成器 → received = "B" → 打印 "Received: B"
```

> ⚠️ **send() 的返回值**：`g.send(data)` 的返回值是生成器 **下一次 yield 的值**（不是本次发送的 data）！

### 🖥️ 第17题 — 装饰器链执行顺序 ⭐⭐⭐

**输出：**
```
[A] before
[B] before
Hello
[B] after
[A] after
```

**解析（装饰器链的执行顺序）**：
```python
# @make_decorator("A")
# @make_decorator("B")
# def greet(): ...
# 
# 等价写法：
# greet = make_decorator("A")(make_decorator("B")(greet))
#         ↑外层装饰器        ↑内层装饰器

# ① 进入 A 的 wrapper → 打印 "[A] before"

# ④ B 的 wrapper 中调用原始 greet() → 打印 "Hello"
# ⑤ 回到 B 的 wrapper → 打印 "[B] after"
# ⑥ 回到 A 的 wrapper → 打印 "[A] after"
```

**记忆口诀**：**"穿衣服"比喻** — 先穿内衣(B)，再穿外套(A)；脱衣服时先脱外套(A)，再脱内衣(B)。

```python
import time
import functools

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
```

### ✍️ 第19题（8分）— 带参数装饰器：重试机制 ⭐⭐⭐

**参考实现**：
```python
import time
import functools

def retry(times=3, delay=1):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, times + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"尝试 {attempt}/{times} 失败: {e}")
                    if attempt < times:
                        time.sleep(delay)
                    else:
                        print("所有重试均失败，抛出异常")
                        raise   # 最后一次失败，重新抛出异常
        return wrapper
    return decorator

# 使用示例：
@retry(times=3, delay=0.5)
def unstable_network_call():
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络不稳定")
    return "成功"

print(unstable_network_call())
```

**得分点**：
- ✅ 三层嵌套结构正确（2分）
- ✅ 捕获异常并实现重试逻辑（2分）
- ✅ 全部失败时重新抛出异常（2分）
- ✅ 使用 `functools.wraps`（1分）
- ✅ 支持任意参数（1分）

### ✍️ 第20题（8分）— 类装饰器：单例模式 ⭐⭐⭐

**参考实现**：
```python
class Singleton:
    def __init__(self, cls):
        self.cls = cls
        self.instance = None
    
    def __call__(self, *args, **kwargs):
        if self.instance is None:
            print(f"创建 {self.cls.__name__} 的唯一实例")
            self.instance = self.cls(*args, **kwargs)
        else:
            print(f"返回已有的 {self.cls.__name__} 实例")
        return self.instance

@Singleton
class DatabaseConnection:
    def __init__(self, host):
        self.host = host

# 测试：
db1 = DatabaseConnection("localhost")
db2 = DatabaseConnection("localhost")
print(db1 is db2)   # True（同一个实例）
print(db1.host)     # localhost
```

**更优雅的实现（使用 `__new__` 方法，不是类装饰器但更常用）**：
```python
class Singleton:
    _instance = None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

# 测试：
s1 = Singleton()
s2 = Singleton()
print(s1 is s2)   # True
```

### 🎯 第23题（10分）— 装饰器链实战 ⭐⭐⭐

**参考实现**：
```python
# 全局路由表
url_routes = {}

def route(path):
    """路由装饰器：把函数注册到路由表"""
    def decorator(func):
        url_routes[path] = func
        print(f"注册路由: {path} → {func.__name__}")
        return func   # 注意：这里直接返回原函数（不包装），只是做注册
    return decorator

def get_handler(path):
    """根据路径查找处理函数"""
    handler = url_routes.get(path)
    if handler:
        return handler
    else:
        raise ValueError(f"404 Not Found: {path}")

# 使用示例：
@route("/hello")
def hello_handler():
    return "Hello, World!"

@route("/about")
def about_handler():
    return "About Page"

# 测试：
print(url_routes)
# {'/hello': hello_handler, '/about': about_handler}

handler = get_handler("/hello")
print(handler())   # Hello, World!

handler = get_handler("/about")
print(handler())   # About Page
```

### 🏆 第25题 — LC #509 斐波那契数（生成器版）

**思路**：
```python
def fibonacci_generator():
    """无限斐波那契生成器"""
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

def fibonacci(n):
    """获取第 n 个斐波那契数（0-indexed）"""
    gen = fibonacci_generator()
    for _ in range(n + 1):
        result = next(gen)
    return result

# 测试：
print(fibonacci(0))   # 0
print(fibonacci(1))   # 1
print(fibonacci(10))  # 55
```
