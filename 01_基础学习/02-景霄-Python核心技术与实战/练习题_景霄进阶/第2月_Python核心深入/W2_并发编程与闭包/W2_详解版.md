### 🔘 第1题 — GIL概念 ⭐

**正确答案: B** ✅ → **多线程程序的并行执行（CPU密集型）**

> **GIL（Global Interpreter Lock，全局解释器锁）**：
> - CPython（标准Python解释器）的内存管理**不是线程安全**的
> - GIL确保同一时刻只有一个线程在执行Python字节码
> - **影响**：多线程无法利用多核CPU加速CPU密集型任务
> - **不影响的**：IO密集型任务（线程在IO时会释放GIL）

| 任务类型 | 多线程效果 | 原因 |
|---------|-----------|------|
| CPU密集型 | ❌ 无加速，甚至可能更慢 | GIL限制 |
| IO密集型 | ✅ 有加速 | IO等待时释放GIL |

### 🔘 第4题 — multiprocessing ⭐⭐

**正确答案: B** ✅ → **可以绕过GIL，实现真正的多核并行**

```python
# 每个进程有独立的GIL，互不干扰
import multiprocessing as mp

def cpu_intensive(x):
    return sum(i*i for i in range(x))

if __name__ == "__main__":
    with mp.Pool(processes=4) as pool:   # 4个进程，利用4核CPU
        results = pool.map(cpu_intensive, [10**7, 10**7, 10**7, 10**7])
```

> 📌 **Pool常用方法**：
> - `map(func, iterable)`：同步映射（返回结果列表）
> - `imap()`：惰性版本（内存更优）
> - `apply_async()`：异步执行单个任务

### 🔘 第7题 — 闭包概念 ⭐

**正确答案: B** ✅ → **捕获了 `x`**

```python
def outer():
    x = 10               # ← 被inner捕获的变量
    def inner():
        print(x)         # ← 引用了外层函数的变量x（形成闭包）
    return inner

f = outer()              # f 是 inner 函数，且携带了 x=10 的环境
f()                      # 输出: 10
```

> 📌 **闭包定义**：内部函数引用了外部函数的变量，即使外部函数已执行完毕，变量依然可被访问。

### 🔘 第8题 — LEGB规则 ⭐⭐

**正确答案: A** ✅ → **Local → Enclosing → Global → Built-in**

```python
# LEGB 查找顺序：
# L - Local：当前函数内部
# E - Enclosing：外层函数（嵌套函数）
# G - Global：模块全局
# B - Built-in：Python内置

x = "global"

def outer():
    x = "enclosing"
    def inner():
        # x = "local"   # 如果取消注释，输出 "local"（L）
        print(x)          # 查找顺序：L → E → G → B
    inner()

outer()                  # 输出: "enclosing"（E）
```

### 🔘 第9题 — nonlocal关键字 ⭐⭐

**正确答案: C** ✅ → **`1` 和 `2`**

```python
def outer():
    x = 0
    def inner():
        nonlocal x        # ← 声明：x 不是局部变量，而是外层的x
        x += 1
        return x
    return inner

counter = outer()
print(counter())          # x=0 → x+=1 → return 1  输出: 1
print(counter())          # x=1 → x+=1 → return 2  输出: 2
```

> ⚠️ **易错点**：如果去掉 `nonlocal x`，会报错 `UnboundLocalError`（Python认为x是局部变量，但没赋值就使用了）！

### 🔘 第10题 — 闭包经典陷阱 ⭐⭐⭐

**正确答案: B** ✅ → **`2 2 2`**

```python
funcs = []
for i in range(3):
    def f():
        return i          # ← 闭包捕获的是变量 i 本身（引用），不是当前值！
    funcs.append(f)

# 循环结束后，i = 2
# 所有 f() 引用的是同一个 i，值为 2
for f in funcs:
    print(f())            # 2 2 2
```

**修复方法（使用默认参数捕获当前值）**：
```python
funcs = []
for i in range(3):
    def f(i=i):          # ← 默认参数在定义时求值并保存
        return i
    funcs.append(f)

for f in funcs:
    print(f())            # 0 1 2  ✅
```

### 🖥️ 第16题 — nonlocal vs global ⭐⭐⭐

**输出：**
```
1
2
0
```

**解析**：
```python
count = 0                      # 全局变量

def counter_factory():
    count = 0                  # 局部变量（和全局count不同！）
    def counter():
        nonlocal count          # ← 声明：使用外层的count（不是全局的）
        count += 1
        return count
    return counter

c = counter_factory()
print(c())                     # count=0→1，输出: 1
print(c())                     # count=1→2，输出: 2
print(count)                    # 全局count依然是0，输出: 0
```

> ⚠️ **`global` vs `nonlocal`**：
> - `global`：声明使用全局变量
> - `nonlocal`：声明使用外层函数的变量（不是全局）

### ✍️ 第18题（10分）— 多线程爬虫模拟 ⭐⭐

**参考实现**：
```python
import threading
import time

def crawl(url):
    """模拟爬取网页"""
    print(f"开始爬取: {url}")
    time.sleep(1)              # 模拟IO等待
    print(f"完成爬取: {url}")

def multi_thread_crawl(urls):
    threads = []
    start = time.time()
    
    # 创建并启动线程
    for url in urls:
        t = threading.Thread(target=crawl, args=(url,))
        threads.append(t)
        t.start()
    
    # 等待所有线程完成
    for t in threads:
        t.join()
    
    print(f"总耗时: {time.time() - start:.2f}秒")

# 测试：
urls = ["http://example.com/1", "http://example.com/2", "http://example.com/3"]
multi_thread_crawl(urls)
# 输出：总耗时约 1秒（而不是3秒）✅
```

**得分点**：
- ✅ 使用 `threading.Thread` 创建线程（3分）
- ✅ `start()` 启动线程（2分）
- ✅ `join()` 等待所有线程完成（3分）
- ✅ 验证总耗时约1秒（2分）

```python
import asyncio
import time

async def fetch(url):
    """模拟异步爬取"""
    print(f"开始爬取: {url}")
    await asyncio.sleep(1)      # 模拟IO等待（非阻塞！）

async def async_crawl(urls):
    start = time.time()

    return results

# 测试：
urls = ["http://example.com/1", "http://example.com/2", "http://example.com/3"]
results = asyncio.run(async_crawl(urls))
print(results)
# 输出：总耗时约 1秒 ✅
```

### ✍️ 第20题（12分）— 闭包实现计数器 ⭐⭐

**参考实现**：
```python
def make_counter(initial=0):
    """闭包实现计数器工厂"""
    count = initial
    
    def counter():
        nonlocal count
        current = count
        count += 1
        return current
    
    def reset(new_initial=0):
        nonlocal count
        count = new_initial
    
    # 可选：给计数器添加方法
    counter.reset = reset
    return counter

# 测试：
c1 = make_counter(0)
c2 = make_counter(10)

print(c1())                     # 0
print(c1())                     # 1
print(c2())                     # 10
print(c2())                     # 11
print(c1())                     # 2（c1和c2独立）✅

# 进阶版：支持多个操作
def make_advanced_counter(initial=0):
    count = initial
    
    def increment():
        nonlocal count
        count += 1
        return count
    
    def decrement():
        nonlocal count
        count -= 1
        return count
    
    def get():
        return count
    
    def reset(new_val=0):
        nonlocal count
        count = new_val
    
    return {
        "inc": increment,
        "dec": decrement,
        "get": get,
        "reset": reset
    }

counter = make_advanced_counter(0)
print(counter["inc"]())         # 1
print(counter["inc"]())         # 2
print(counter["dec"]())         # 1
print(counter["get"]())         # 1
```

### 🎯 第21题（10分）— GIL理解与应对 ⭐⭐⭐

**参考答案**：

**1. 什么是GIL？为什么Python（CPython）要有GIL？**

- **GIL（Global Interpreter Lock）**：全局解释器锁，是一把互斥锁，确保同一时刻只有一个线程在执行Python字节码。
- **为什么要有GIL？**
  - CPython的内存管理（引用计数）**不是线程安全**的
  - 如果多个线程同时修改引用计数，会导致内存泄漏或崩溃
  - GIL是一种简单有效的解决方案（虽然牺牲了多核并行）

**2. GIL对多线程程序的影响是什么？**

- **CPU密集型任务**：多线程**无法加速**，甚至可能更慢（上下文切换开销）
- **IO密集型任务**：多线程**可以有效加速**（IO等待时会释放GIL）
- **单线程程序**：不受影响

**3. 如何应对GIL的限制？（至少说出3种方法）**

| 方法 | 适用场景 | 原理 |
|------|---------|------|
| `multiprocessing` | CPU密集型 | 多进程，每个进程独立GIL |
| `asyncio` | IO密集型（高并发） | 协程，单线程事件循环 |
| C扩展（如NumPy） | 科学计算 | C代码释放GIL |
| PyPy（无GIL版本） | 通用 | 换解释器 |
| 多进程 + 协程 | 混合任务 | 进程池 + 协程 |

### 🎯 第22题（10分）— 并发模型选择 ⭐⭐

**参考答案**：

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 1. 爬取100个网页（网络IO密集） | `asyncio` 或 `threading` | IO密集型，asyncio更高并发；threading也可 |
| 2. 计算100万个素数的个数（CPU密集） | `multiprocessing` | CPU密集型，需要绕过GIL，利用多核 |
| 3. 高并发Web服务器（大量网络连接） | `asyncio` | 大量IO等待，协程开销最小（比线程小得多） |
| 4. 批量处理图片（CPU密集+IO密集混合） | `multiprocessing` + `asyncio` | CPU密集用多进程，IO密集用协程 |

**混合模型示例**：
```python
import asyncio
import concurrent.futures

def cpu_intensive_task(n):
    """CPU密集型任务"""
    return sum(i*i for i in range(n))

async def main():
    # 使用ProcessPoolExecutor处理CPU密集型任务
    with concurrent.futures.ProcessPoolExecutor() as executor:
        loop = asyncio.get_event_loop()
        # 在线程池中执行CPU密集型任务（不阻塞事件循环）
        result = await loop.run_in_executor(executor, cpu_intensive_task, 10**7)
        print(result)

asyncio.run(main())
```

## 🏆 LeetCode 本周预备题 — 解题思路

### 🏆 第23题 — LC #1114 按序打印（并发基础）

**题目要求**：三个函数 `first()`、`second()`、`third()` 按序执行（first → second → third）。

**使用 Lock 实现**：
```python
import threading

class Foo:
    def __init__(self):
        self.lock1 = threading.Lock()
        self.lock2 = threading.Lock()
        self.lock1.acquire()   # 锁定lock1（second必须等待）
        self.lock2.acquire()   # 锁定lock2（third必须等待）
    
    def first(self, printFirst):
        printFirst()
        self.lock1.release()   # 释放lock1，允许second执行
    
    def second(self, printSecond):
        self.lock1.acquire()   # 等待first释放lock1
        printSecond()
        self.lock2.release()   # 释放lock2，允许third执行
    
    def third(self, printThird):
        self.lock2.acquire()   # 等待second释放lock2
        printThird()
```

**使用 Event 实现（更简洁）**：
```python
import threading

class Foo:
    def __init__(self):
        self.event1 = threading.Event()
        self.event2 = threading.Event()
    
    def first(self, printFirst):
        printFirst()
        self.event1.set()       # 通知second可以执行
    
    def second(self, printSecond):
        self.event1.wait()     # 等待first的信号
        printSecond()
        self.event2.set()      # 通知third可以执行
    
    def third(self, printThird):
        self.event2.wait()     # 等待second的信号
        printThird()
```
